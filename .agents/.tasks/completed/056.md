# Remove Redundant StreamRecord Success Booleans

**Source:** prompt.md item 25

**Priority:** P1

**Effort:** M

**Problem:** Stream task completion records encode both `success` and `is_error`, which represent the same bit and can drift.

**Files involved:** `src/clients/types.rs`, `src/clients/agent.rs`, docs/tests for stream-json

**Fix:** Keep one canonical outcome field internally. Preserve external compatibility if needed through a serializer shim or versioned stream format.

**Status:** Completed

**Completed:** New task completion records serialize only the canonical `is_error`
outcome flag. Legacy records with `success` still deserialize and are validated
against the subtype.
