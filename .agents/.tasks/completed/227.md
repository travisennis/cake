# Decide LCOV missing-source warning policy

## Problem

Coverage currently emits a warning like `N source files had no matching entry in the LCOV report` for every extracted `*_tests.rs` file. Those files produce no LCOV entry, which is why they must be excluded from CRAP scoring.

The warning is easy to miss. A new extracted test file with a complex helper can increase CRAP noise until someone notices the warning and updates the exclusion behavior.

## Relevant Files

- `scripts/check-coverage.sh` - top-level coverage gate.
- `scripts/cargo-crap.sh` - CRAP wrapper and exclusions.
- Extracted Rust test modules matching `**/*_tests.rs`.

## Fix Direction

After the extracted-test exclusion behavior is fixed, decide whether the missing-source warning should still be actionable:

- If a glob exclusion makes extracted test files fully moot, document or assert that behavior and leave the warning non-fatal.
- If the warning can still indicate unscored source that should be covered or excluded intentionally, promote it to a hard error with clear remediation.

## Acceptance Notes

- [x] The team has an explicit policy for LCOV source files with no matching coverage entry.
  **Decision**: The warning is non-fatal. `*_tests.rs` files contain only test-only code
  with no instrumented coverage data, so they are expected to have no LCOV entries.
  The policy is documented in `scripts/check-coverage.sh` with a comment and
  `--ignore-filename-regex '_tests\\.rs$'` added to the `cargo llvm-cov --lcov`
  invocation to suppress the warning for these files.
- [x] Extracted `*_tests.rs` files no longer create a silent CRAP-scoring trap.
  (Completed by task 226 — glob `--exclude '**/*_tests.rs'` in cargo-crap.sh.)
- [x] If the warning remains non-fatal, the task records why that is safe after the exclusion fix.
  The comment in `check-coverage.sh` explains: extracted test modules have no LCOV
  entries because they contain only test-only code, and they are already excluded from
  CRAP scoring via `--exclude '**/*_tests.rs'` in `scripts/cargo-crap.sh`.
- [x] If the warning becomes fatal, the output tells maintainers exactly how to resolve it.
  Not applicable — the warning remains non-fatal. This check is preserved as
  documentation of the decision.
