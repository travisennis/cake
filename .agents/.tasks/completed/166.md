# Suppress Session File Path When Sessions Are Disabled

Created: 2026-05-21

## Summary

When cake is invoked with `--no-session`, output metadata can still include a
`session_file` path. This is misleading because the user explicitly requested
that the invocation not save a session, and in observed runs the printed path
did not correspond to a persisted JSONL session file.

The issue was observed while running cake from `/Users/travisennis/Projects/acai-ts`
with `--no-session --output-format json` during task 165 investigation. The JSON
response included fields like:

```json
{
  "session_file": "/Users/travisennis/.local/share/cake/sessions/<uuid>.jsonl",
  "session_id": "<uuid>"
}
```

Attempts to inspect that `session_file` path afterward failed because the file
did not exist.

## Relevant Areas

- `src/main.rs`
- `src/config/session.rs`
- `src/types/session.rs`
- Session creation and output formatting for `--output-format json` and
  `--output-format stream-json`

## Fix Direction

Clarify the runtime contract for `--no-session` output:

- Do not print or serialize a concrete `session_file` path when session
  persistence is disabled.
- Preserve any session/task identifiers that are intentionally still used for
  in-memory tracing or stream records, but make sure persisted-file metadata is
  absent or explicitly null when no file is written.
- Cover text, JSON, and stream JSON output paths if they expose session file
  metadata.

## Acceptance Criteria

- Running cake with `--no-session --output-format json` does not report a
  session file path that will not be written.
- Running cake with normal session persistence still reports the correct
  session file path where applicable.
- Add focused regression coverage for the affected output formatting or session
  metadata path.
- Run the required Rust checks for a CLI/session behavior change, including
  `cargo check --tests`, `cargo fmt`, and `just ci`.
