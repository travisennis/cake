# Simplify Agent Turn Item Processing

## Summary

Agent::send currently loops over returned turn items multiple times, then re-borrows the appended history tail to recover function calls. Extract function call data before moving items into history and combine the display/progress pass where practical without changing stream or progress ordering.

## Source

Captured from `agent-loop-perf-review.md`, "Real wins" item 4.

## Problem

After `complete_turn()` returns, `Agent::send()` currently:

1. Scans `items` to detect function calls.
2. Streams every item.
3. Scans `items` again for progress reporting.
4. Moves `items` into conversation history.
5. Re-borrows the newly appended history tail and scans again to collect function-call triples.

The final re-scan is only needed because the code moves `items` before preparing tool execution data.

## Relevant Files

- `src/clients/agent.rs` - `Agent::send()` turn loop
- `src/clients/agent_runner.rs` - `TurnResult` boundary, if helper extraction needs it
- Agent loop tests under `src/clients/agent.rs` and related client test modules

## Fix Direction

Extract the owned function-call data needed by hook and tool execution before moving `items` into history. Merge the stream/progress passes only if doing so keeps ordering obvious and preserves the existing final assistant-message skip behavior.

This is a refactor task: do not change tool execution concurrency, hook ordering, conversation history order, streamed JSON shape, or CLI progress semantics.

## Acceptance Notes

- [x] `Agent::send()` no longer re-borrows the just-appended history tail to discover function calls.
- [x] Stream and progress callbacks preserve current ordering and final assistant-message behavior.
- [x] Existing agent loop tests pass (725 tests); the stream/progress passes remain separate loops so no ordering pinning was needed.
- [x] Ran `cargo check --tests`, `cargo fmt`, and `just ci` — all pass.

## Implementation Notes

Extracted owned `(String, String, String)` function-call data from `items` before moving into history, removing the need to re-borrow from the just-appended history tail. Derived `has_tool_calls` from the extracted data. Simplified the hook future closure by eliminating redundant `.to_owned()` calls since the values are already owned.
