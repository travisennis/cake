# Fix agent turn change-risk regression

## Problem

CI still fails in the Coverage job's `Check change-risk regression` step after the false-positive cargo-crap exclusions were aligned. One remaining production regression is `CodingAssistant::execute_agent_turn` in `src/main.rs`.

Current GitHub output from run `28116394203`:

- `CodingAssistant::execute_agent_turn` CRAP 16.9, delta +0.1, CC 15, coverage 79.5%

This function coordinates hook execution, prompt context records, task start/complete records, agent send, success handling, and error handling. The fix must improve tested behavior without changing agent-loop semantics.

- `src/main.rs`
- `src/clients/agent.rs`

## Fix Direction

Add focused coverage around the missing branches of `execute_agent_turn`, especially hook/no-hook, success/error, stop hook context, error hook execution, and task completion record outcomes.

If direct testing is awkward, introduce the smallest behavior-preserving helper extraction needed to make branches testable. Avoid broad agent-loop refactors.

Do not reduce coverage thresholds or exclude this function from cargo-crap analysis.

## Acceptance Notes

- Extracted `handle_agent_turn_result` from `execute_agent_turn` (line 690+), reducing the latter's cyclomatic complexity from ~15 to ~3. The extracted function contains all the post-send branching (match on send outcome, stop hook, error hook, task complete record).
- 5 new unit tests cover every branch of `handle_agent_turn_result`:
  - Success without hooks
  - Success with empty hooks (stop returns None)
  - Success with stop hook returning additional context (real shell command)
  - Error without hooks
  - Error with empty hooks (error_occurred called)
- All tests run without hitting external APIs — they inject controlled `Result` values directly into the extracted handler.
- `execute_agent_turn` now delegates to the extracted helper; its remaining branches (hook/no-hook pre-send path, `send()` call) are structurally simpler and benefit from existing integration coverage.
- Agent-loop, hook, session-record, and stream-json compatibility surfaces are preserved.
- `cargo test` passes (758 tests, all pass). `cargo clippy --all-targets` is clean.
- The cargo-crap baseline will need regeneration (`just change-risk-baseline` after this is merged) so that the new function's expected CRAP is captured and the old function's reduced CRAP is reflected.
