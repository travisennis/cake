# Update Session Evaluation Skills For Current Task Completion Schema

**Created:** 2026-05-17
**Source:** Evaluation of session `c3593cfb-907f-40b1-be04-cd78d07a9f8f`

## Problem

The session evaluation guidance still refers to a `success` field on `task_complete`, but current v4 session records use `subtype` and `is_error`.

This caused evaluation tooling to query `success` and get `null` even though the record clearly showed successful completion.

## Evidence

In session `c3593cfb-907f-40b1-be04-cd78d07a9f8f`:

- Line 208 contains `{"type":"task_complete","subtype":"success","is_error":false,...}`.
- The evaluation query from the skill selected `{task_id, subtype, success, duration_ms, ...}` and produced `"success": null`.
- `docs/design-docs/session-management.md` already documents `task_complete.subtype` and `task_complete.is_error` as the current schema.

## Relevant Files

- `.agents/skills/session-investigation/SKILL.md`
- `.agents/skills/evaluating-cake/SKILL.md`
- `.agents/skills/debugging-cake/SKILL.md` if it has stale examples or queries

## Fix Direction

Update the project skills to match the current v4 session schema:

- Replace `success` field references in task-complete queries with `subtype` and `is_error`.
- Describe success as `subtype == "success"` and `is_error == false`.
- Keep notes about legacy or unsupported session formats separate from current v4 guidance.
- Prefer examples that include `tool_call_count` when summarizing task outcomes.

Do not change the persisted session schema for this task; the solution is to update the skills and examples.

## Acceptance Notes

- Session investigation/evaluation examples no longer select a nonexistent `success` field for v4 records.
- Guidance consistently uses `subtype` and `is_error` for task outcome.
- Debugging guidance remains compatible with current session docs.
- Markdown is reviewed for accuracy; full CI is not required unless code or config changes are introduced.
