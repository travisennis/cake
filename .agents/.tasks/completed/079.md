# Introduce RetryPolicy Configuration Type

**Source:** prompt.md item 53

**Priority:** P2

**Effort:** M

**Problem:** Retry behavior is encoded in module-level public constants. This makes retry policy hard to test or eventually configure.

**Files involved:** `src/clients/retry.rs`, settings model if configured

**Fix:** Add a `RetryPolicy` struct with defaults and pass it into retry decision logic. Keep current defaults unchanged.

**Status:** Completed

**Completion notes:** Added `RetryPolicy` with unchanged default retry budget and backoff values. Retry classification and delay calculation now take the policy explicitly, `AgentRunner` owns a default policy for current runtime behavior, and focused retry tests cover custom policy budget/backoff behavior.
