# Remove Progress Spinner from Text Output Mode

## Problem

When `--output-format text` (the default), cake uses `indicatif::ProgressBar`
to show a spinner with status messages ("Thinking...", "Bash: rg ...",
"Responding...") during agent turns. This progress feature:

1. Causes visual noise and terminal state pollution — the spinner writes escape
   sequences to stderr that persist after the run, cluttering logs and CI output.
2. Introduces a dependency on `indicatif` (and its transitive deps like
   `console`) that exists only for this single use case.
3. Adds complexity to `CliOutputSink`, `Agent`, and `AgentObserver` via
   progress/retry callback plumbing that has no other consumer.

The JSON and stream-JSON output modes already skip progress entirely.

## Relevant Files

- `src/cli/output.rs` — `CliOutputSink::attach_text_progress()`,
  `finish_progress()`, `format_spinner_message()`, `format_retry_message()`,
  uses `indicatif::{ProgressBar, ProgressStyle}`
- `src/clients/agent.rs` — `with_progress_callback()`,
  `with_retry_callback()`, `report_progress()`
- `src/clients/agent_observer.rs` — `ProgressCallback`, `RetryCallback`,
  `set_progress_callback()`, `set_retry_callback()`, `report_progress()`,
  `report_retry()`
- `src/clients/agent/agent_loop.rs` — calls `report_progress()` and
  `report_retry()` during the loop
- `Cargo.toml` — `indicatif = "0.18"` dependency
- `src/clients/retry.rs` — `RetryStatus` and `RetryReason` types (may need
  trimming if retry callbacks are unused after)

## Fix Direction

1. Remove the progress bar/spinner setup from `CliOutputSink` — delete
   `attach_text_progress()` and `finish_progress()` (or simplify it to no-op).
   Make `attach_callbacks()` return `(Agent, None)` for text mode.
2. Remove the `ProgressCallback` and `RetryCallback` types and their
   infrastructure from `AgentObserver`, `Agent`, and the agent loop.
3. Remove the `indicatif` dependency from `Cargo.toml`.
4. Delete or simplify `format_spinner_message()` and `format_retry_message()`
   if they are unused after removal.
5. Check the agent loop to ensure no remaining references to progress/retry
   reporting.
6. Update any tests that reference the progress infrastructure.

- [x] `indicatif` is removed from `Cargo.toml` and `Cargo.lock`.

- [x] All tests pass (`cargo test`).
- [x] No dead code or dangling references to removed types.

## Changes Made

- **`src/cli/output.rs`**: Removed `indicatif` imports, `attach_text_progress()`,
  `finish_progress()`, `format_spinner_message()`, `format_retry_message()`, and
  `format_done_summary()`. Simplified `attach_callbacks()` to return `Agent` (no
  `Option<ProgressBar>`).
- **`src/clients/agent_observer.rs`**: Removed `ProgressCallback`, `RetryCallback`
  types and their `set_*`/`report_*` methods. Stripped `progress`/`retry` fields.
- **`src/clients/agent.rs`**: Removed `with_progress_callback()` and
  `with_retry_callback()` methods and `report_progress()` private helper.
- **`src/clients/agent/agent_loop.rs`**: Removed `report_progress()` and
  `report_retry()` calls. Simplified `stream_and_report_turn_items()` to
  `stream_turn_items()` (streaming only). Removed unused `Role` import.
- **`src/clients/agent_runner.rs`**: Removed `report_retry` callback parameter
  from `complete_turn()` and `wait_for_retry()`. Removed stale
  `too_many_arguments` expect.
- **`src/clients/agent/agent_tests.rs`**: Updated `test_429_rate_limit_retries_and_succeeds`
  to not use `with_retry_callback()`.
- **`src/clients/tools/*`**: Removed dead `summarize_args()` functions from
  `bash.rs`, `edit.rs`, `read.rs`, `write.rs`. Removed `ToolSummarizer` type,
  `summarize` field from `ToolEntry`, `summarize()` from `ToolRegistry`.
  Removed `summarize_tool_args()` and `truncate_display()` from `mod.rs`.
  Removed now-unnecessary `summarize_args_only_requires_path` test.
- **`src/clients/mod.rs`**: Removed `summarize_tool_args` re-export.
- **`src/main.rs`**: Removed spinner/retry/done-summary test functions and
  their imports. Simplified `handle_interrupt()` (no spinner parameter).
  Removed unused `Duration` and `ConversationItem` imports.
- **`src/time_format.rs`**: Removed unused `format_duration_tenths()` which
  was only called by the removed `format_retry_message()`.
- **`src/cli/mod.rs`**: Removed `#[cfg(test)]` re-exports of removed functions.
- **`Cargo.toml`**: Removed `indicatif = "0.18"` dependency.
- **`ci/cargo-crap-baseline.json`**: Regenerated to account for removed code.
