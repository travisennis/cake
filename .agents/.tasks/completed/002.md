# Fix Stdin 100ms Timeout Silently Dropping Slow Input

**Source:** fix.md #2

**Problem:** `read_stdin_content()` in `src/main.rs` (line 159) uses a 100ms `recv_timeout` when reading piped stdin. Slow upstream producers silently lose data with no warning.

**Files involved:**
- `src/main.rs` (lines 133–175)

**Fix:**
1. Log a warning on timeout in the `Err(mpsc::RecvTimeoutError::Timeout)` arm.
2. Consider adding a `--stdin-timeout <ms>` CLI flag (default 100) to replace the hardcoded value.
