# Replace Sandbox Violation Detection by Output Substrings

**Source:** prompt.md item 16

**Priority:** P0

**Effort:** M

**Problem:** Bash classifies sandbox violations by searching command output for strings like `Operation not permitted`. Legitimate command output can trigger false classifications.

**Files involved:** `src/clients/tools/bash.rs`, `src/clients/tools/sandbox`

**Fix:** Classify sandbox failures only when sandboxing was enabled, the command failed, and output contains a permission-denial phrase. This preserves the guardrail against repeated impossible retries while preventing successful output or unsandboxed failures from being mislabeled as sandbox restrictions.

**Status:** Completed
