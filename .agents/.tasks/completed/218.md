# Fix macOS sandbox change-risk regressions

## Problem

CI still fails in the Coverage job's `Check change-risk regression` step after the false-positive cargo-crap exclusions were aligned. Several remaining production regressions are in macOS sandbox detection and probe reporting.

Current GitHub output from run `28116394203`:

- `MacOsSandbox::probe_can_apply_profile` CRAP 15.4, delta +4.4, CC 8, coverage 51.3%
- `detect_platform` CRAP 5.3, delta +1.9, CC 3, coverage 36.4%
- `MacOsSandbox::profile_probe_failure` CRAP 2.0, delta +1.0, CC 1, coverage 0.0%
- `SandboxProfileProbe::unavailable` CRAP 2.0, delta +1.0, CC 1, coverage 0.0%

This is sandbox-sensitive code. The fix must not weaken fail-closed behavior.

## Relevant Files

- `src/clients/tools/sandbox/macos.rs`
- `src/clients/tools/sandbox/mod.rs`
- Sandbox-related tests
- `docs/guardrails/sandboxing-and-filesystem-boundaries.md`
- `docs/design-docs/sandbox.md`

## Fix Direction

Add focused tests or small test seams for macOS sandbox probe outcomes without depending on the host runner's real sandbox state. Cover:

- probe unavailable with failure details
- profile probe failure accessor
- `detect_platform` fail-closed message path
- any helper extraction needed for deterministic tests

Preserve sandbox compatibility surfaces and call out security impact in the handoff.

Do not reduce coverage thresholds, bypass sandbox checks, or exclude these production paths from cargo-crap analysis.
