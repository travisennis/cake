# Document and Enforce Dead-Code Suppression Policy

**Status:** Completed
**Priority:** P1
**Effort:** S
**Created:** 2026-05-14
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

Commit `3cc2e73` handled dead-code warnings inconsistently: some sites became `#[cfg(test)]`, some `#[expect(dead_code, reason = ...)]`, some were correctly identified as stale annotations and removed. There is no written policy, so future contributors and coding agents will keep diverging.

## Files Involved

- `AGENTS.md` (Code Style Guidelines section)
- All current `#[expect(dead_code, ...)]` and `#[cfg(test)]` annotations introduced by `3cc2e73`

## Fix Direction

1. Add a short subsection to `AGENTS.md` under "Code Style Guidelines" with this policy:
   - **Default**: delete dead code.
   - **Acceptable**: `#[cfg(test)]` for items that are genuinely only useful from `#[cfg(test)]` callers (test fixtures, test-only constructors).
   - **Last resort**: `#[expect(dead_code, reason = "...")]` only for serde fields that must exist for deserialization to succeed but are not read by application logic. The reason must say so explicitly.
   - Forbidden: `#[allow(dead_code)]` (already enforced by `allow_attributes`).
2. Sweep current `#[expect(dead_code, ...)]` and `#[cfg(test)]` sites and reconcile to the policy. In particular, audit:
   - The five `#[expect(dead_code, reason = "API response field preserved for completeness")]` annotations in `src/clients/chat_types.rs` and `src/clients/types.rs` (handled by task 130).
   - Any remaining `#[cfg(test)]` on `pub fn` items.
3. Confirm `just ci` passes.
