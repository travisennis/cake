## Problem

Cake's system prompt encourages broad parallel tool use with guidance like "If you can call multiple tools in one turn, do so." That is useful for reads and independent checks, but it is unsafe guidance for multiple `Edit` or `Write` calls that target the same file.

In session `a91aa043-4833-4118-88ff-92feef8688b9`, the model followed the parallelism guidance and issued several same-file `Edit` calls in one turn:

- Lines 69-70: two edits for `src/config/model.rs`.
- Lines 76-81: six edits for `src/config/settings.rs`.
- Lines 104-109: six edits for `src/config/data_dir.rs`.

The session then showed lost edits and corrupted trailing code. Task 163 should add runtime protection, but the model-facing prompt should also teach the intended behavior.

## Relevant Files

- `src/prompts.rs`
- Tool descriptions for `Edit` and `Write`
- Tests or snapshots for generated system prompt/tool schemas

## Fix Direction

Update prompt/tool guidance so it distinguishes safe parallelism from unsafe mutating overlap:

- Encourage parallel `Read` calls and independent non-mutating `Bash` checks.
- Allow parallel `Edit`/`Write` only when target paths are distinct and the edits do not depend on each other.
- Instruct the model not to issue multiple `Edit` or `Write` calls for the same file in one turn.
- Tell the model to re-read and edit sequentially when a file has just been changed or when exact text may have shifted.

Keep the wording concise so it does not bloat every session.

## Acceptance Notes

- The generated prompt/tool descriptions explicitly warn against same-file parallel `Edit`/`Write` calls.
- Existing efficiency guidance still encourages parallel reads and independent operations.
- Tests or snapshots are updated where prompt text is asserted.
