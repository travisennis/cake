# Improve macOS Sandbox Probe Diagnostics and Cache Documentation

**Source:** prompt.md items 50 and 51

**Priority:** P3

**Effort:** M

**Problem:** `probe_can_apply_profile` caches capability for the process lifetime and returns only a boolean, so later user-facing errors cannot include the concrete probe failure details. The cache may be acceptable because macOS Seatbelt profile applicability is effectively a process-context property, but that rationale is not documented. The original probe scratch-file cleanup concern appears to have been addressed by `tempfile::Builder::tempfile_in`.

**Files involved:** `src/clients/tools/sandbox/macos.rs`, `src/clients/tools/sandbox/mod.rs`

**Fix:** Preserve useful probe failure information, such as spawn errors, exit status, stderr, and stdout fallback, so `detect_platform` can produce a more actionable macOS sandbox unavailable error. Add an explicit comment or small helper-level documentation explaining why process-lifetime caching is intentional. Do not rework per-command sandbox profile lifetime in this task; if that cleanup policy needs changing, track it separately.

**Acceptance notes:**

- Probe failure details are available to the error path that reports macOS sandbox unavailability.
- The process-lifetime cache has an explicit rationale in code comments or nearby documentation.
- The existing probe temp-file cleanup behavior remains based on `tempfile` drop semantics.
