# Reduce Cloning in Agent Tool Loop

**Source:** prompt.md item 4

**Priority:** P2

**Effort:** M

**Depends on:** 048 (e2e test as safety net)

**Problem:** The agent tool loop clones call fields and skill location maps more than necessary, including cloning a whole `HashMap` per concurrent tool future.

**Files involved:** `src/clients/agent.rs`

**Fix:** Move owned function call data where possible and wrap shared immutable maps in `Arc`. Keep behavior unchanged and use tests to guard concurrency behavior.

**Status:** Completed

**Completion notes (2026-05-11):**
- Shared immutable skill location metadata with `Arc<HashMap<_, _>>` so concurrent tool futures no longer clone the full map.
- Reworked the agent tool loop to borrow function call fields from the appended turn history for pre-tool hook planning, then move owned hook plans into execution futures.
- Preserved concurrent hook/tool execution behavior and validated with `cargo test clients::agent`.
