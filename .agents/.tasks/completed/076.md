# Optimize Edit Application Allocation

**Source:** prompt.md item 38

**Priority:** P2

**Effort:** M

**Problem:** `apply_edits_reverse_order` rebuilds a full `String` for every edit, creating avoidable allocation and copy costs.

**Files involved:** `src/clients/tools/edit.rs`

**Fix:** Use `replace_range` on a single mutable string while applying reverse-sorted edits, or build the output in one pass.
