# Introduce ToolContext (no plumbing yet)

**Parent:** 047

**Source:** prompt.md items 1 and 62

**Priority:** P1

**Effort:** S

**Depends on:** 048 (e2e test as safety net)

**Problem:** Tools read cwd, temp dirs, settings dirs, skill dirs, and additional dirs from five separate process-global `OnceLock` values. There is no single type that represents "the directory context a tool runs in."

**Files involved:** `src/clients/tools/mod.rs`, `src/main.rs`

**Fix:**
1. Define a `ToolContext` struct that holds: `cwd: PathBuf`, `temp_dirs: Vec<PathBuf>`, `additional_dirs: Vec<PathBuf>`, `skill_dirs: Vec<PathBuf>`, `settings_dirs: Vec<PathBuf>`.
2. Add a constructor that builds it from the same sources currently used by the `OnceLock`s.
3. Construct one `ToolContext` in `main.rs` at startup. Continue feeding the existing `OnceLock`s from it for now (no behavior change).
4. Add unit tests confirming `ToolContext` construction is deterministic and independent of process state.

**Status:** Completed
