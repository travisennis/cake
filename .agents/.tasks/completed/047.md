# Plumb ToolContext Through Tool Execution (Parent)

**Source:** prompt.md items 1 and 62

**Priority:** P1

**Effort:** XL (split into 047a / 047b / 047c)

**Depends on:** 048 (e2e test as safety net)

**Problem:** Tools read cwd, temp dirs, settings dirs, skill dirs, and additional dirs from process-global `OnceLock` values. This prevents isolated tool contexts and leaks state across tests or multiple agents.

This task is too large to land as one change without blocking other refactors. It is split into three sequential subtasks:

- **047a — Introduce ToolContext** — define the type, populate it from current globals at startup. No behavior change.
- **047b — Plumb &ToolContext through tool execution** — pass it through validation, tool execution, and sandbox profile construction. Depends on 051 (Tool abstraction) so the trait can carry the context.
- **047c — Delete the OnceLocks** — remove `ADDITIONAL_DIRS`, `SKILL_DIRS`, `SETTINGS_DIRS`, `CWD`, `TEMP_DIRS` and their setters/getters once nothing else reads them.

**Files involved:** `src/clients/tools/mod.rs`, tool modules, `src/clients/agent.rs`, `src/main.rs`

**Status:** Completed

**Completion notes:** 047a introduced `ToolContext`, 047b plumbed it through tool execution and sandbox construction, and 047c removed the legacy process-global directory caches.
