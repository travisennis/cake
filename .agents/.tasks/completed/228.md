# Make check-coverage failures diagnosable

## Problem

`scripts/check-coverage.sh` currently combines two independent checks behind one exit path:

- the total coverage threshold, currently 90%
- the cargo-crap regression check

When `check-coverage` fails, the output does not make it obvious which gate fired. That slows down triage for CI and local `just ci` failures.

## Relevant Files

- `scripts/check-coverage.sh` - top-level coverage gate that combines total coverage and cargo-crap checks.
- `scripts/cargo-crap.sh` - regression check invoked by `check-coverage`.
- CI / `just ci` output that surfaces the failing command.

## Fix Direction

Make failures diagnosable at a glance. Either separate the checks into distinct invocations/commands or keep the combined command but emit clear section headers and failure summaries that identify whether the failure came from total coverage, cargo-crap regression, or both.

Preserve existing CI semantics unless deliberately changed: meaningful total-coverage failures and meaningful cargo-crap regressions should still fail.

## Acceptance Notes

- [x] A total-coverage threshold failure clearly says the total coverage gate failed.
- [x] A cargo-crap regression failure clearly says the cargo-crap regression gate failed.
- [x] If both checks fail, output makes both failures visible.
- [x] Verification covers representative success and failure paths, using tests, script fixtures, or documented manual commands.

## Verification

```bash
# Both gates fail (real state: coverage ~89.96% < 90%, regression exists):
scripts/check-coverage.sh

# Coverage passes, regression gate fails (lower threshold):
COVERAGE_THRESHOLD=80 scripts/check-coverage.sh

# Both gates pass (hypothetical):
COVERAGE_THRESHOLD=80 CRAP_REGRESSION_EPSILON=999 scripts/check-coverage.sh
```

Actual output (June 2026 state, both gates failing):
```

FAIL: CRAP regression detected (exit code 1)

FAIL: One or more coverage gates failed
```
