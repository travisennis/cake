# Clean Up macOS Sandbox String Construction Style

**Source:** prompt.md item 71

**Priority:** P3

**Effort:** M

**Depends on:** 040 (Seatbelt profile builder)

**Problem:** macOS sandbox profile code mixes `String::new`, `.to_string()`, and `format!` patterns heavily, making rule construction noisy.

**Files involved:** `src/clients/tools/sandbox/macos.rs`

**Fix:** After introducing a profile builder, simplify string construction and remove redundant conversions.

**Status:** Completed

## Outcome

Replaced the `SeatbeltProfileBuilder::deny_default()` constructor that mixed `String::new()`, `.to_string()` on string literals, and a `vec![]` initializer with clean builder-method calls:

- Added `SeatbeltProfileBuilder::new()` (const fn, empty vec) and builder methods `version()` and `deny_default()` so every profile line goes through a builder method.
- Removed redundant inner `use std::io::Write` import in `write_profile_to_temp` (shadowed module-level import).
- Replaced `to_string_lossy().to_string()` with `to_string_lossy().into_owned()` in `apply` for idiomatic `Cow<str>` → `String` conversion.
