# Type Hook Matcher Sources

**Parent:** 060

**Source:** prompt.md item 48

**Priority:** P1

**Effort:** S

**Problem:** `HookRunner::payload(...)` and the matcher-source plumbing dispatch hook events using a typed `HookEvent` enum on one side and a free-form `Option<&str>` source on the other. The matcher source for tool events is the tool name as a string. Misspelling or renaming a tool silently breaks the matcher contract.

**Files involved:** `src/hooks.rs`, `src/config/hooks.rs`

**Fix:**
1. Replace `Option<&str> source` arguments with a typed `HookSource` enum: `HookSource::None`, `HookSource::Tool(ToolName)`, `HookSource::SessionStart(SessionStartReason)`, etc.
2. Use the same enum in matcher loading from settings, so a misspelled matcher source is a settings-load error rather than a silent no-match.
3. Add tests covering matcher-source serialization round-trips.

**Status:** Completed
