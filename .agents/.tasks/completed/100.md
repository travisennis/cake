# Decide Linux Landlock Default Feature Policy

**Source:** prompt.md item 78

**Priority:** P3

**Effort:** M

**Problem:** `landlock` is optional and not enabled by default, so Linux builds may lack the expected OS sandbox unless the feature is selected.

**Files involved:** `Cargo.toml`, sandbox docs, CI

**Fix:** Decide whether Landlock should be enabled by default on Linux or explicitly documented as opt-in. Update features, docs, and CI coverage.

**Decision:** Official Linux release artifacts should include Landlock support by building the Linux release target with `--features landlock`. Cargo's global default feature set stays empty so the Linux-only dependency remains explicit for source builds and downstream packagers. See `docs/adr/006-linux-landlock-release-artifacts.md`.

**Implementation notes:** Update `.github/workflows/release.yml` so only the Linux release target passes `--features landlock`. Keep CI's existing `--all-features` and `--no-default-features` coverage. Update README and sandbox design docs to distinguish official release artifacts from source builds.

**Status:** Completed
