# Avoid Duplicate User Message Construction

**Source:** src/clients/agent.rs review

**Problem:** `Agent::send()` constructs the same user `ConversationItem::Message` twice: once for history and once for streaming.

**Files involved:**
- `src/clients/agent.rs:353`
- `src/clients/agent.rs:365`

**Fix:** Build one user conversation item, stream it, then push it into history. Clone only where required by ownership.
