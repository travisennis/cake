# Make Hooks Observable in Typed Session Flow (Parent)

**Source:** prompt.md items 47, 48, and 49

**Priority:** P1

**Effort:** L (split into 060a / 060b / 060c)

**Problem:** Hook behavior uses partially documented optional booleans, string event/source matching, and a deserialized `_suppress_output` field. The API is hard to reason about.

This is split into three behaviorally distinct fixes that can land independently:

- **060a — Type hook matcher sources** — replace string event/source matching in `payload(...)` and dispatch with typed enums.
- **060b — Model continue/block as proper enum** — replace `Option<bool> r#continue` + `decision: Option<String>` + `permission: Option<String>` with a single typed `HookDecision` enum with documented semantics.
- **060c — Implement or remove suppress_output handling** — `HookOutput::_suppress_output` is deserialized then ignored. Either honor it (and document where it takes effect) or remove the field entirely.

**Files involved:** `src/hooks.rs`, `src/config/hooks.rs`, `src/clients/types.rs`, hook docs

**Status:** Completed

All child tasks complete (060a, 060b, 060c).
