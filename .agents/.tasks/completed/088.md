# Clean Up Dead Code Allows

**Source:** prompt.md item 24

**Priority:** P3

**Effort:** S

**Problem:** Some public methods or conversion helpers are marked with `#[allow(dead_code)]`, hiding whether they are intentionally reserved or simply unused.

**Files involved:** `src/clients/agent.rs`, `src/clients/types.rs`

**Fix:** Delete unused code or add comments and tests for intentionally retained APIs.

**Status:** Completed

**Resolution:** No `#[allow(dead_code)]` annotations remain in the target files. The only remaining dead-code suppression in the project was `#[expect(dead_code)]` on `HookGroup::source_path` in `src/config/hooks.rs`, which violated the project policy (expect is reserved for serde fields only). Removed the unused field and its annotation from `HookGroup`; adjusted all construction sites in `src/config/hooks.rs`, `src/hooks.rs` (tests), and `src/clients/agent.rs` (tests). The project now has zero `#[allow(dead_code)]` or non-conforming `#[expect(dead_code)]` annotations.
