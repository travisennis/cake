# Fail on Missing Function Call Fields in Responses API

**Source:** prompt.md items 14 and 43

**Priority:** P0

**Effort:** S

**Problem:** Responses API parsing uses `unwrap_or_default()` for function call `id`, `call_id`, `name`, and `arguments`. Missing identifiers become empty strings and can collide downstream.

**Files involved:** `src/clients/responses.rs`, `src/clients/types.rs`

**Fix:** Treat missing required function call fields as malformed API responses. Return an error with enough context for debugging, and update tests.

**Status:** Completed
