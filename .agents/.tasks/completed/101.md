# Review Module Size and Ownership Boundaries

**Source:** prompt.md module size critique

**Priority:** P3

**Effort:** XL

**Status:** Completed

**Problem:** Several files exceed the 800 production-line threshold or have excessive inline test code. This makes ownership and review harder.

**Thresholds:**
- **Production split**: files with >800 non-test lines
- **Test extraction**: files with >800 test lines AND >40% test ratio
- Tests extracted to sibling `_tests.rs` files via `#[path = "..."] mod tests;`

**Files involved:** large Rust modules across `src/`

- Minimal public API churn; no behavior changes
