# Add Ctrl-C Handling With Graceful Shutdown

**Created:** 2026-06-09
**Source:** Code review of `src/main.rs` and the agent loop (2026-06-09)

## Problem

cake installs no signal handler (`tokio::signal` is unused anywhere in the
crate). When the user interrupts a run with Ctrl-C:

- The session file ends without a `TaskComplete` record, so consumers of the
  stream/session format see a task that started and never finished.
- `emit_session_summary_telemetry` never runs, leaving the telemetry sidecar
  without a summary record.
- Worktree cleanup (`cleanup_worktree` in `src/main.rs`) is skipped, leaving
  the temporary worktree registered and the original directory not restored.
- Hook lifecycle events (`stop`/`error_occurred`) never fire.

Bash tool children are already covered (`kill_on_drop(true)` in
`src/clients/tools/bash.rs:313`), so the gap is the session/task lifecycle,
not orphaned subprocesses.

## Relevant Files

- `src/main.rs` — `main()`, `CodingAssistant::run`, `execute_agent_turn`
- `src/clients/agent/agent_loop.rs` — `Agent::send` loop
- `src/types/` — `TaskOutcome` (may need an `Interrupted` variant)
- Task 178 (worktree RAII guard) — a `Drop`-based guard would also cover the
  Ctrl-C path for worktrees

## Fix Direction

- Listen for `tokio::signal::ctrl_c()` and race it against the agent turn
  (e.g., `tokio::select!` around `execute_agent_turn`, or a cancellation token
  checked between turns/tool batches).
- On interrupt, emit a task completion record with an interrupted outcome,
  write the telemetry summary, run worktree cleanup, and exit with a
  distinct exit code.
- Decide whether a second Ctrl-C should force immediate exit.
- An `Interrupted` task outcome changes the session/stream record contract, so
  check whether an ADR is needed before implementation.

## Acceptance Notes

- Interrupting a run mid-turn produces a session file whose final task record
  reflects the interruption (no dangling `TaskStart`).
- Worktrees created with `--worktree` are cleaned up (or kept-with-changes)
  on interrupt, matching the normal-exit policy.
- A second Ctrl-C (if supported) forces exit without hanging.
- Tests cover the interrupted-outcome record shape.
- `just ci` passes.
