# Fix HTTP Client Build Error Handling

**Source:** prompt.md item 3

**Priority:** P1

**Effort:** S

**Problem:** `build_http_client` falls back to `reqwest::Client::default()` when builder construction fails, silently discarding timeout and connection-pool settings.

**Files involved:** `src/clients/agent.rs`

**Fix:** Propagate the builder error through `Agent::new` or use an explicit `expect` with a clear invariant if the builder cannot fail with current settings.

**Status:** Completed

**Resolution:** Replaced the silent `reqwest::Client::default()` fallback with an explicit invariant panic so HTTP client builder failures can no longer discard timeout and connection-pool settings unnoticed.

**Verification:** `cargo test clients::agent`; `just ci`
