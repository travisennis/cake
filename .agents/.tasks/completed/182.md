# Derive Copy For Usage Counters

## Summary

Usage and nested token-detail structs contain only u64 counters but are Clone-only. Derive Copy for these value types and replace unnecessary clone calls where touched.

## Source

Captured from `agent-loop-perf-review.md`, "Real wins" item 5.

## Problem

`Usage`, `InputTokensDetails`, and `OutputTokensDetails` are plain counter structs made only of `u64` values. They are cloned in agent turn accounting, task completion records, and telemetry paths even though copying them is semantically correct and cheaper to express.

## Relevant Files

- `src/types/usage.rs`
- `src/clients/agent_runner.rs`
- `src/clients/agent.rs`
- `src/session_telemetry.rs`

## Fix Direction

Derive `Copy` for all three usage structs alongside `Clone`. Remove `.clone()` calls where the surrounding code becomes clearer, but avoid a broad mechanical churn pass outside usage handling.

## Acceptance Notes

- [x] `Usage`, `InputTokensDetails`, and `OutputTokensDetails` derive `Copy`.
- [x] Obvious usage-counter `.clone()` calls in agent accounting and telemetry are replaced with copies.
- [x] Serialization/deserialization behavior is unchanged.
- [x] Run `cargo test usage`, `cargo check --tests`, `cargo fmt`, and `just ci`.
