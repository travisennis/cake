# Correlate Hook Event Records With Tool Calls

**Created:** 2026-05-17
**Source:** Evaluation of session `c3593cfb-907f-40b1-be04-cd78d07a9f8f`

## Problem

Persisted `hook_event` records are useful audit data, but they are hard to correlate with the tool call that triggered them. Session analysis currently has to infer the relationship from ordering.

In session `c3593cfb-907f-40b1-be04-cd78d07a9f8f`, 73 `hook_event` records were stored. They include event name, source, command, exit code, duration, decision, stdout, and stderr, but not the triggering tool call's `call_id`.

## Evidence

- Line 89 recorded a `PostToolUse` hook from `./.cake/hooks/rust-command-failure-context.sh` with useful additional context after a failed `just ci`, but the relationship to Bash call `call_238b` is only inferable from nearby line ordering.
- Line 26 recorded a `PreToolUse` hook whose stdout was `{ "decision": "allow" }`, while the top-level persisted `decision` was `"none"`. Analysis has to inspect raw stdout to understand whether the hook allowed, blocked, or made no effective decision.

## Relevant Files

- `src/config/hooks.rs`

- `docs/design-docs/hooks.md`

## Fix Direction

Persist enough metadata on each `hook_event` record to make session analysis direct:

- Include the triggering tool call id, such as `call_id` or `tool_use_id`, for `PreToolUse`, `PostToolUse`, and failure hook events.
- Include the tool name and a compact command/input summary where useful.
- Store the resolved hook decision separately from raw hook stdout so an analyzer does not have to parse stdout to know the effective decision.
- Keep backward compatibility for older session files without these fields.

Because this changes persisted session audit shape, check whether an ADR or design-doc update is needed before implementation.

## Acceptance Notes

- New hook event records can be directly joined to the associated `function_call` by id.
- Hook records still preserve raw stdout/stderr for debugging, subject to existing caps.
- The effective hook decision is clear without reparsing raw stdout.
- Session documentation describes the new fields and backward compatibility.
- Tests or snapshots cover hook events associated with at least one tool call.
- `just ci` passes.
