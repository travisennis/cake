# Centralize Config Directory Resolution and Honor XDG_CONFIG_HOME

**Created:** 2026-06-09
**Source:** Code review of `src/main.rs` and `src/config/settings.rs` (2026-06-09)

## Problem

The config directory `~/.config/cake` is independently hardcoded in two
places:

- `src/main.rs:761` ‚Äî `dirs::home_dir().unwrap_or_else(...).join(".config").join("cake")`,
  used for system prompt resolution.
- `src/config/settings.rs:375` ‚Äî the same construction for
  `settings.toml` loading.

Neither honors `$XDG_CONFIG_HOME`. If the two sites ever drift, the system
prompt and settings would load from different config roots. (`data_dir.rs`
also builds `~/.config/AGENTS.md` paths independently.)

## Relevant Files

- `src/main.rs` ‚Äî config dir construction in `run`
- `src/config/settings.rs` ‚Äî `SettingsLoader` global path
- `src/config/data_dir.rs` ‚Äî XDG-adjacent AGENTS.md lookups
- `src/prompts/mod.rs` ‚Äî consumes the config dir

## Fix Direction

Add a single `config_dir()` helper in the config module and use it from both
sites. Consider resolving via `$XDG_CONFIG_HOME` falling back to
`~/.config` (the `dirs` crate's `config_dir()` differs on macOS ‚Äî it returns
`~/Library/Application Support` ‚Äî so an explicit XDG-style helper preserving
current `~/.config/cake` behavior is likely the right call). Document the
resolution order; configuration shape is a compatibility surface, so keep
default behavior unchanged unless explicitly deciding otherwise.

## Acceptance Notes

- Created `src/config/config_dir.rs` with a single `config_dir()` helper that:
  - Checks `$XDG_CONFIG_HOME` first (if set and non-empty)
  - Falls back to `~/.config` (or `./.config` as last resort)
  - All call sites go through this one function
- Updated all four call sites:
  - `src/main.rs` ‚Äî `config_dir().join("cake")` for session factory config_dir
  - `src/config/settings.rs` ‚Äî `config_dir().join("cake").join("settings.toml")` for global settings
  - `src/config/hooks.rs` ‚Äî `config_dir().join("cake").join("hooks.json")` for global hooks
  - `src/config/data_dir.rs` ‚Äî `config_dir().join("AGENTS.md")` for XDG-level AGENTS.md
- Existing `~/.config/cake` behavior is preserved by default (when `$XDG_CONFIG_HOME` is unset)
- If `$XDG_CONFIG_HOME` is set, it is honored for all cake config resolution
- All 775 tests pass (includes 745 unit, 30 integration)
- `cargo fmt` applied
