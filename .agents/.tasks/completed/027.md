# Clean Up Pre-Existing Dead Code

**Source:** Deslop review of staged v4 session migration changes

**Problem:** Several methods and variants are marked `#[allow(dead_code)]` and have no callers in production. These were pre-existing before the current change and were intentionally left untouched during deslop to avoid scope creep.

**Items to address:**

1. **`Agent::activated_skills()`** — `src/clients/agent.rs:175-180`
   - Marked `#[allow(dead_code)]`, zero callers anywhere.
   - Could be useful as a public API for debugging. Decide whether to keep or remove.

2. **`TaskCompleteSubtype::ErrorMaxTurns`** — `src/clients/types.rs:237`
   - Variant is defined but never constructed. Only `Success` and `ErrorDuringExecution` are ever created.
   - Either wire it up when max turns are exceeded during the agent loop, or remove the variant.

3. **`SessionRecord::to_streaming_json`** — `src/clients/types.rs:653` — Resolved by task 061
   - Marked `#[allow(dead_code)]`. Only called from tests (snapshot tests in same file).
   - Production code serializes `StreamRecord` directly via `serde_json::to_string`. Consider whether this method still serves a purpose or should become test-only.
   - 2026-05-10: Removed while consolidating conversation serialization paths in task 061.

4. **`DataDir::save_session`** — `src/config/data_dir.rs:141`
   - Marked `#[allow(dead_code)]`. Only called from tests in `data_dir.rs`.
   - Production now uses `create_session_file` + live append via `PersistCallback`. The batch-save path may still be useful for utilities or future use cases.

5. **`ConversationItem::to_streaming_json`** — `src/clients/types.rs:168-219` — Resolved by task 061
   - Marked `#[allow(dead_code)]`. Only called from snapshot tests.
   - Production serializes `StreamRecord` directly. The snapshot tests that call this method test code production never uses, giving false confidence.
   - Either remove the method and refactor tests to snapshot `StreamRecord` serialization, or keep it as a test-only documented contract.
   - 2026-05-10: Removed while consolidating conversation serialization paths in task 061; snapshots now cover `StreamRecord::from_conversation_item`.

6. **`SessionFileHeader::session_id`** — `src/config/data_dir.rs:367-368`
   - Field is `#[allow(dead_code)]`, deserialized but never read.
   - Remove the field from the struct; serde ignores unknown fields anyway.

**Status:** Completed — 2026-05-11

**Resolution:**

1. **Agent::activated_skills()** — Kept. Now has test callers in main.rs (skill activation tests added after task was written). Added `#[allow(dead_code)]` to the internal `SkillActivations::active()` helper as well, since it's only used through this path. The method provides useful public API for debugging and testing.

2. **TaskCompleteSubtype::ErrorMaxTurns** — Removed the `ErrorMaxTurns` variant from both `TaskCompleteSubtype` and `TaskOutcome`. No max-turns concept exists in the agent loop, and the variant was never constructed in production or tests. Cleaned up all match arms and serde serialization/deserialization.

3. **SessionRecord::to_streaming_json** — Already resolved by task 061.

4. **DataDir::save_session** — Kept with `#[allow(dead_code)]`. The method is actively used by 8 test functions in data_dir.rs and provides a clean batch-save API that may be useful for future utilities.

5. **ConversationItem::to_streaming_json** — Already resolved by task 061.

6. **SessionFileHeader::session_id** — Removed the unused field. Serde ignores unknown fields, so existing session files deserialize correctly.
