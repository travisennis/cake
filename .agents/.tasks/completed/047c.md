# Delete the Tool OnceLocks

**Parent:** 047

**Source:** prompt.md items 1 and 62

**Priority:** P1

**Effort:** S

**Depends on:** 047b (no callers reach into globals anymore)

**Problem:** With `ToolContext` plumbed through, the process-global `OnceLock`s in `src/clients/tools/mod.rs` are dead weight. Leaving them in place invites future code to reach for them again.

**Files involved:** `src/clients/tools/mod.rs`, `src/main.rs`

**Fix:**
1. Delete `static CWD`, `static TEMP_DIRS`, `static ADDITIONAL_DIRS`, `static SKILL_DIRS`, `static SETTINGS_DIRS` and their setter/getter helpers (`set_additional_dirs`, `set_skill_dirs`, `set_settings_dirs`, `cached_cwd`, `cached_temp_dirs`, etc.).
2. Update `main.rs` to construct `ToolContext` directly without going through global setters.
3. Remove the now-stale comment about "Uses `OnceLock` instead of `thread_local!`."
4. Confirm `cargo test` and `just ci` pass with no remaining references to the deleted statics.

**Status:** Completed

**Completion notes:** Removed the tool directory `OnceLock` caches and global setters/getters. `main.rs` now keeps the startup `ToolContext` as the single source of tool directory state, and default/test agents build uncached current-process contexts.
