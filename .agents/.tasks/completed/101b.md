# Extract CliOutputSink, RunMode, and Session Construction from main.rs

**Parent:** 101 (Review Module Size and Ownership Boundaries)

**File:** `src/main.rs` (1,377 production lines)

**Goal:** Extract three self-contained units to the `cli/` module.

**What moves:**

1. **`CliOutputSink`** → `src/cli/output.rs` (~150 lines). Struct + impl. Pure rendering, no coupling to `CodingAssistant` fields. Extracted as-is.

2. **`RunMode`** → `src/cli/run_mode.rs` (~65 lines). Enum + impl. Pure session-mode logic. Extracted as-is.

3. **Session construction** → `src/cli/session_factory.rs` (~250 lines). Five methods: `skill_locations`, `new_client_and_session`, `restored_client_and_session`, `forked_client_and_session`, `build_client_and_session`. `skill_locations` converted to free function (no `self` dependency). Remaining four stay as methods via `impl CodingAssistant` block.

**Note:** `CodingAssistant` must become `pub(crate)` to allow impl blocks in `cli/` submodules.

**~465 lines extracted. `main.rs` drops from ~1,377 → ~912 production lines.**

**Acceptance:** `just ci` passes. No behavioral change. All existing tests pass. `cargo build` succeeds with re-exported types in `cli/mod.rs`.
