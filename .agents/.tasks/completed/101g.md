# Add Module Size Lint Script and Just Recipe

**Parent:** 101 (Review Module Size and Ownership Boundaries)

**Goal:** Create a script that checks Rust source files against the production-line and test-line thresholds established in task 101. Include a `just` recipe so it can be run during CI after the subtask refactors land.

**Thresholds:**
- Production lines: >800 → warn
- Test lines: >800 AND test ratio >40% → warn
- Test module detection: `mod tests { }` blocks (brace-counting, not regex)

**Script:** `scripts/lint-module-size.py`

**Output:** Lists each file exceeding a threshold with its counts:
```
src/clients/agent.rs: 2047 production lines (threshold: 800)
src/clients/tools/edit.rs: 1025 test lines, 56% test ratio (threshold: 800 lines / 40%)
```

**Exit code:** 0 (warn-only). The script always succeeds — it's informational. A `--strict` flag can be added later to fail CI once thresholds are met.

**Just recipe:**
```makefile
# Check module sizes against thresholds (informational, always passes)
lint-module-size:
    python3 scripts/lint-module-size.py
```

**Acceptance:**
- `just lint-module-size` runs and lists current violations
- Script correctly distinguishes production lines from test module lines
- The recipe is not wired into `just ci` yet (will be added after 101a–101f complete)
- Add a TODO comment in the `ci` recipe where it should be inserted
