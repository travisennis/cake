# Introduce a Seatbelt Profile Builder

**Source:** prompt.md item 52

**Priority:** P0

**Effort:** L

**Problem:** macOS sandbox profiles are assembled through many `Vec<String>` pushes and `format!` calls. Escaping and rule shapes are duplicated and easy to get wrong.

**Files involved:** `src/clients/tools/sandbox/macos.rs`

**Fix:** Add a small profile builder with methods for literal paths, subpaths, and common rule types. Make the builder own escaping and update profile generation to use it.

**Status:** Completed

## Outcome

- Added a `SeatbeltProfileBuilder` that centralizes profile line construction, path matchers, regex rules, target rules, and path escaping.
- Migrated macOS sandbox profile generation and helper sections to use builder methods for literal paths, prefixes, subpaths, and common allow rules.
- Kept raw rule insertion limited to non-path Seatbelt forms that do not need filesystem path escaping.
