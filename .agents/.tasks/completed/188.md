# Add Duplicate Mutation Guard Coverage for Write and Hook Blocks

**Created:** 2026-05-29
**Source:** Review of commit `b4e765140df9`

## Problem

Task 163 added duplicate same-file mutation rejection after pre-tool hooks and before execution. The current tests cover same-file `Edit`/`Edit`, same-file `Edit`/`Write`, relative-vs-absolute paths, different-file mutations, and repeated `Read` calls.

The suite does not directly cover:

- `Write` followed by `Write` for the same existing file.
- `Write` followed by `Write` for the same non-existent file.
- A hook-blocked mutating call followed by an executable mutating call for the same path.

The hook-block case is important because the guard intentionally skips non-`Execute` hook plans when building the seen-path set.

## Relevant Files

- `src/clients/agent.rs`
- `src/clients/tools/write.rs`
- `src/clients/tools/edit.rs`

## Fix Direction

Add focused duplicate-guard tests using the existing `duplicate_guard` test pattern. At minimum, assert that the second same-path `Write` is rejected and that a hook-blocked first call does not cause a later executable same-path mutation to be rejected.

Include a non-existent target case if the `Write` target extraction canonicalizes or validates it in a way that can support the scenario without weakening production validation.

## Acceptance Notes

- Same-turn `Write`/`Write` duplicate rejection is covered.
- Hook-blocked mutating calls are covered and do not populate the duplicate guard's seen set.
- Run `cargo test duplicate_guard`, `cargo check --tests`, `cargo fmt`, and the required final verification for test changes.
