# Flatten Agent send() Result by Removing the Unreachable None

**Created:** 2026-06-09
**Source:** Code review of `src/clients/agent/agent_loop.rs` (2026-06-09)

## Problem

`Agent::send` returns `anyhow::Result<Option<String>>`
(`src/clients/agent/agent_loop.rs:59`), but `Ok(None)` is unconstructable:
the only success return is
`Ok(Some(self.conversation.resolve_assistant_message()))`, and
`resolve_assistant_message` (`src/clients/agent_state.rs:122`) always returns
a `String`, synthesizing placeholder text when the response is incomplete.

The dead `Option` has already bred unreachable code downstream:

- `src/cli/output.rs:129` — the "No response received from the model" warning
  branch can never run.
- `src/cli/output.rs:164` — `response_text.as_deref().unwrap_or("")` never
  takes the `""` arm.
- `TurnResult.result` (`src/cli/output.rs:21`) and hook plumbing
  (`runner.stop(result_text.as_deref())`) carry the `Option` through layers.

## Relevant Files

- `src/clients/agent/agent_loop.rs` — `send`
- `src/clients/agent_state.rs` — `resolve_assistant_message`
- `src/cli/output.rs` — `TurnResult`, `render_text_result`,
  `turn_result_json`
- `src/main.rs` — `execute_agent_turn`
- `src/hooks.rs` — `stop` hook signature if it takes `Option<&str>`

## Fix Direction

Change `send` to return `anyhow::Result<String>` and remove the `Option`
through `TurnResult`, the output sink, and the stop hook plumbing. If the
distinction "real assistant message vs. synthesized placeholder" matters to
any consumer, model it explicitly (e.g., an enum) instead of an `Option` that
is never `None` — but prefer the simple flatten unless a consumer needs it.
