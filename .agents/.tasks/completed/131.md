# Fix `SandboxConfig` `struct_field_names` Reason and Verify Dropped `dead_code`

**Status:** Completed
**Priority:** P2
**Effort:** S
**Created:** 2026-05-14
**Completed:** 2026-05-15
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

Commit `3cc2e73` converted `#[allow(clippy::struct_field_names, dead_code)]` on `SandboxConfig` into `#[expect(clippy::struct_field_names, reason = "field names mirror the struct name for clarity")]`. Two problems:

1. **Wrong reason**: the lint fires because of the shared `read_*` prefix (`read_write`, `read_only`), not because field names mirror the struct name (the struct is `SandboxConfig`).
2. **Dropped `dead_code`**: the original `#[allow(... dead_code)]` was load-bearing on platforms where these fields are not consumed by any cfg branch. Need to verify whether non-macOS, non-Linux builds (the `#[cfg(not(any(target_os = "macos", target_os = "linux")))]` branch) still compile.

## Files Involved

- `src/clients/tools/sandbox/mod.rs` (around line 36-44)
- `src/clients/tools/sandbox/macos.rs`
- `src/clients/tools/sandbox/linux.rs`

## Implementation

Renamed fields to eliminate the shared `read_*` prefix that triggered `clippy::struct_field_names`:

- `read_write` → `writable`
- `read_only_exec` → `system_paths`
- `read_only` → `readable`

Removed the `#[expect(clippy::struct_field_names)]` entirely since the lint no longer fires.

For `dead_code` on unsupported platforms (not macOS, not Linux), restructured by adding explicit cfg-gated field reads in `build_with_additional_dirs()` rather than using `#[expect(dead_code)]`, which is reserved for serde fields per project code style.

Updated all references across `mod.rs`, `macos.rs`, and `linux.rs` including test code and struct literals.

## Verification

- `cargo clippy --all-targets --all-features` clean
- All 608 tests pass
- Linux landlock paths (with/without feature) use renamed fields and remain consistent
- Unsupported-platform dead_code handled via explicit field reads (no cross-compile target available)
