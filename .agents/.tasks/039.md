# Escape Home Paths in macOS Sandbox Profiles

**Source:** prompt.md item 15

**Priority:** P0

**Effort:** M

**Problem:** macOS Seatbelt profile generation interpolates `$HOME` into profile strings inconsistently. Some rules do not use `escape_path`, so unusual path characters can break the profile.

**Files involved:** `src/clients/tools/sandbox/macos.rs`

**Fix:** Route all path interpolation through one escaping helper. Add tests for quotes, backslashes, spaces, and parentheses in home/config paths.

**Status:** Completed

## Outcome

- HOME-derived macOS Seatbelt profile rules now build paths with `Path` joining and route interpolation through the same escaping helper used by config paths.
- Git, SSH config, gh/glab, and user Keychain rules now escape quotes and backslashes from HOME before inserting paths into profile strings.
- Added regression coverage for HOME paths containing quotes, backslashes, spaces, and parentheses.
