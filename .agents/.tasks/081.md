# Replace Retry Signal Body Substring Matching Where Possible

**Source:** prompt.md item 55

**Priority:** P2

**Effort:** M

**Problem:** Retry classification relies on provider response body substring checks for some conditions, which can be brittle across providers.

**Files involved:** `src/clients/retry.rs`, backend error handling

**Fix:** Prefer structured error fields, status codes, and provider-specific classifiers. Keep substring fallback isolated and logged.

**Status:** Completed

**Completion notes:** `src/clients/retry.rs` now classifies overloaded-provider retry signals from structured JSON error `type`/`code` fields first. Body substring matching is limited to malformed or plain-text responses and logs when that fallback is used. Added focused regression tests for structured fields, valid JSON messages that mention `overloaded_error`, and the isolated unstructured fallback. Updated `docs/design-docs/api-retry-strategy.md` to document the structured-first classifier.
