# Add Backend Abstraction for API Providers

**Source:** prompt.md item 5

**Priority:** P1

**Effort:** L

**Depends on:** 048 (e2e test as safety net)

**Problem:** Agent code dispatches on `ApiType` and calls backend-specific request and response functions directly. Adding a backend requires touching orchestration code.

**Files involved:** `src/clients/agent.rs`, `src/clients/responses.rs`, `src/clients/chat_completions.rs`

**Fix:** Introduce a backend trait or enum object that owns send/parse behavior. Keep the agent loop independent from backend selection details.

**Status:** Completed

**Completed:** Added a backend abstraction in `src/clients/backend.rs` that owns provider-specific request sending and response parsing for Responses API and Chat Completions. `Agent` now selects a backend during construction and calls through it in `complete_turn`, removing direct `ApiType` dispatch from the orchestration loop.
