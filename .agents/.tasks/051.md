# Add Tool Abstraction and Registry

**Source:** prompt.md item 70

**Priority:** P1

**Effort:** L

**Depends on:** 048 (e2e test as safety net)

**Problem:** Tools are represented by separate definition functions plus match arms for execute and summarize. Adding a tool requires updating multiple stringly-typed locations.

**Files involved:** `src/clients/tools/mod.rs`, tool modules, `src/clients/agent.rs`

**Fix:** Define a tool trait or registry entry containing name, definition, execution, and argument summary behavior. Register Bash, Read, Edit, and Write through one path.

**Status:** Completed

**Completed:** Added a `ToolRegistry`/`ToolEntry` abstraction that keeps each tool's API definition, execution function, and argument summarizer in a single registration path. `Agent` now owns the registry, derives enabled tool names and API definitions from it, and executes tool calls through the registry instead of a standalone match arm. `summarize_tool_args` now uses the same registry-backed summarization path.
