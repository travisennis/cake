# Fix `#[expect(unused_variables)]` on `LandlockSandbox::apply` Will Break Linux Builds

**Status:** Completed
**Priority:** P0
**Effort:** XS
**Created:** 2026-05-14
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

The conversion from `#[allow(unused_variables)]` to `#[expect(unused_variables, reason = "parameter required by trait, unused when landlock feature is disabled")]` on `LandlockSandbox::apply` is incorrect. `#[expect(...)]` requires the lint to actually fire; `#[allow(...)]` did not. In both feature configurations the variables are read, so the expectation is unfulfilled and the build will fail on Linux.

## Files Involved

- `src/clients/tools/sandbox/linux.rs` (lines 101-132)

## Observed Behavior

- With `feature = "landlock"`: `command` is used via `command.pre_exec(...)` and `config` is used via `config.clone()`.
- Without `feature = "landlock"`: the explicit `let _ = config; let _ = command;` reads them.

In both cases the `unused_variables` lint does not fire, so `#[expect(unused_variables)]` will produce a "lint expectation unfulfilled" error on Linux.

The build currently passes on macOS only because `linux.rs` is gated behind `#[cfg(target_os = "linux")]` and is not compiled on Darwin.

## Fix Direction

Delete the `#[expect(unused_variables, ...)]` attribute entirely. No `allow` or `expect` is needed; both cfg branches use the parameters.

## Acceptance Notes

- The attribute is removed from `LandlockSandbox::apply` in `src/clients/tools/sandbox/linux.rs`.
- `cargo clippy --target x86_64-unknown-linux-gnu` (or equivalent Linux build, with and without `--features landlock`) succeeds without warnings.
- `just ci` passes on macOS.
- A note is added to the future Linux CI matrix (or an existing tracking task) so this kind of regression is caught locally.

## Completion Notes

- Removed the unfulfilled `#[expect(unused_variables)]` from `LandlockSandbox::apply`; both cfg branches already use the parameters.
- Updated the GitHub Actions clippy job to run an Ubuntu matrix for both `--all-features` and `--no-default-features`, so the Landlock-enabled and Landlock-disabled Linux branches are checked in CI.
- Local Linux-target clippy attempts were blocked by the missing `x86_64-linux-gnu-gcc` cross compiler required by `aws-lc-sys`; the existing `just clippy-linux` recipe reports that prerequisite explicitly.
