# Fail on Missing Chat Completion Response IDs

**Source:** prompt.md item 14

**Priority:** P0

**Effort:** S

**Problem:** Chat Completions parsing defaults a missing response id to an empty string. Empty identifiers make tracing and session records ambiguous.

**Files involved:** `src/clients/chat_completions.rs`

**Fix:** Decide which identifiers are required by the backend contract. Fail loudly for missing required IDs, or generate an explicit local ID with a clear prefix if the provider legitimately omits one.

**Status:** Completed

## Outcome

- Chat Completions parsing now rejects responses that omit the required top-level response `id` or provide it as an empty string.
- Reasoning and assistant message items now use a validated response ID instead of silently defaulting to an empty identifier.
- Added parser and HTTP response coverage for missing IDs.

- `just ci` passed.
