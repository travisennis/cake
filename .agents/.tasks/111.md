# Round-Trip Test ConversationItem → StreamRecord → SessionRecord

**Source:** Review of commit `21fb640f74a3` (task 061)

**Priority:** P1

**Effort:** S

**Problem:** Task 061 removed the dead hand-built `to_streaming_json` and `to_session_json` helpers in favor of `StreamRecord::from_conversation_item` plus `impl From<StreamRecord> for SessionRecord`. Those two hand-written conversions are now load-bearing for both `--output-format stream-json` and on-disk session persistence, but no test asserts that round-tripping `ConversationItem` through them preserves every field. A future variant addition or a typo (for example dropping `encrypted_content`, `id`, or `summary`) would silently lose data persisted to disk and would not fail any current snapshot.

**Files involved:** `src/clients/types.rs` (tests module)

**Fix:** Add per-variant round-trip tests of the form:

```rust
let stream = StreamRecord::from_conversation_item(&item);
let session = SessionRecord::from(stream);
let restored = session.to_conversation_item().unwrap();
assert_eq!(item, restored);
```

Cover `Message` (user/assistant/system, with and without `id`/`status`/`timestamp`), `FunctionCall`, `FunctionCallOutput`, and `Reasoning` (with `encrypted_content`, with `content`, and with both). This requires deriving `PartialEq` on `ConversationItem` (and `ReasoningContent` / `ReasoningContentKind` if not already), which is a localized change.

**Acceptance notes:** A failing edit to any conversion arm (e.g., commenting out the propagation of `encrypted_content`) causes a test failure. `just ci` passes.

**Status:** Completed
