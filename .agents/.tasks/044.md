# Add Clap Exclusivity for Session Mode Flags

**Source:** prompt.md item 61

**Priority:** P0

**Effort:** S

**Problem:** `--continue`, `--resume`, and `--fork` are mutually exclusive by behavior but not enforced by clap. If combined, runtime branch order silently chooses one.

**Files involved:** `src/main.rs`

**Fix:** Add a clap `ArgGroup` or explicit conflicts so mixed session mode flags produce a CLI parse error. Add CLI tests. Also treat `--no-session` as incompatible with restore/fork modes because it asks the CLI not to persist the session being continued or forked.

**Status:** Completed

**Resolution:** Added a clap `session_mode` argument group covering `--continue`, `--resume`, `--fork`, and `--no-session` so invalid combinations fail during CLI parsing instead of falling through runtime branch order.

**Verification:** `cargo test --test exit_codes`; `just ci`
