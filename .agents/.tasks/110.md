**Status:** Pending

## Summary

Two external articles (Schwartz 2026, Levin 2026) recommend stricter Clippy configurations, especially useful as guardrails for coding agents. The project already enables `pedantic` and `nursery` categories but does not enable `restriction`, has no `clippy.toml` for test allowances, and lacks the `allow_attributes` / `allow_attributes_without_reason` guardrail lints. This task evaluates and applies the most impactful recommendations.

## Gaps to Address

1. **Evaluate enabling the `restriction` category** (Levin's approach) or selectively enabling specific high-value lints from it (Schwartz's approach). See research file for the full priority-ordered list.

2. **Add a `clippy.toml`** with test allowances:
   ```toml
   allow-unwrap-in-tests = true
   allow-expect-in-tests = true
   allow-dbg-in-tests = true
   allow-indexing-slicing-in-tests = true
   allow-panic-in-tests = true
   ```

3. **Enable `allow_attributes` and `allow_attributes_without_reason`** to require `#[expect(lint, reason = "...")]` instead of silent `#[allow]`.

4. **Review `deny` vs `warn` strategy** -- consider switching to `warn` + `-D warnings` on CI for better local iteration.

5. **Fix any warnings** produced by newly enabled lints.

## Acceptance Notes

- `Cargo.toml` `[lints.clippy]` section is updated with the chosen approach
- `clippy.toml` exists with appropriate test allowances
- `just clippy-strict` passes with zero warnings
- Task 089 (allow policy) and 107 (missing_docs policy) should be resolved in the same changeset if practical, since they interact with the same configuration
