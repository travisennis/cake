# Report Unknown Responses Output Types

**Source:** prompt.md item 45

**Priority:** P0

**Effort:** S

**Problem:** `parse_output_items` ignores unknown Responses API output `type` values. Provider changes can produce empty or partial turns with no diagnostic.

**Files involved:** `src/clients/responses.rs`

**Fix:** Emit a structured warning for unknown output types, including the type name and response id when available. Consider returning an error for output items that cannot be safely ignored.

**Status:** Completed

**Resolution:** Unknown Responses API output item types now emit a structured tracing warning with response id, output index, output id, and output type. Responses that contain only unknown output item types now return a diagnostic error instead of producing an empty turn; mixed responses continue with recognized items.

**Verification:** `cargo test clients::responses::tests::parse_output_items_unknown_type`; `just ci`
