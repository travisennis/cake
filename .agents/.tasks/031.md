# Fix rm -rf Safety Checks for Multiple Targets

**Source:** prompt.md item 34

**Priority:** P0

**Effort:** M

**Problem:** `check_dangerous_rm` only validates the first positional target after `rm -rf`. A command like `rm -rf /tmp/x ~/important` can pass because only `/tmp/x` is checked.

**Files involved:** `src/clients/tools/bash_safety.rs`

**Fix:** Parse and validate every non-flag target for recursive forced removal. Add regression tests for multiple safe and unsafe targets.
