# Restore `e.print().ok()` With Localized Expect for `unused_result_ok`

**Status:** Pending
**Priority:** P3
**Effort:** XS
**Created:** 2026-05-14
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

`src/main.rs` previously used `e.print().ok();` to deliberately print clap's error and discard the resulting `Result`. Commit `3cc2e73` changed it to `_ = e.print();` to satisfy the new `unused_result_ok` lint. The new form loses the original `.ok()` intent — "deliberately swallowing the error of printing" — and is no clearer than what it replaced.

## Files Involved

- `src/main.rs` (around line 1241)

## Fix Direction

Restore `e.print().ok();` and add a localized `#[expect(clippy::unused_result_ok, reason = "best-effort error printing; we already exit non-zero")]` immediately above the line.

## Acceptance Notes

- `e.print().ok();` is restored.
- A narrow `#[expect(clippy::unused_result_ok, ...)]` justifies the pattern at the call site.
- `just ci` passes.
