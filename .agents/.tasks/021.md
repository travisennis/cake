# Deduplicate Session Streaming Helpers

**Source:** src/clients/agent.rs review

**Problem:** `stream_record()` and `stream_item()` both serialize a `SessionRecord`, invoke the streaming callback, and push to the stream buffer. The duplicated logic makes future streaming changes easier to miss.

**Files involved:**
- `src/clients/agent.rs:256`
- `src/clients/agent.rs:266`

**Fix:** Have `stream_item()` convert the item with `SessionRecord::from_conversation_item()` and delegate to `stream_record()`.
