# Replace Module-Wide `string_slice` Suppression in retry.rs

**Status:** Pending
**Priority:** P1
**Effort:** S
**Created:** 2026-05-14
**Parent:** 124

## Summary

`src/clients/retry.rs` has a top-of-file `#![expect(clippy::string_slice, reason = "all string indexing in context overflow detection uses indices from .find() which return char boundaries")]`. Replace with localized per-site `#[expect]` attributes (or refactor to UTF-8-safe alternatives) so the lint remains active for any future code added to this file.

## Files Involved

- `src/clients/retry.rs`

## Fix Direction

1. Remove the module-level `#![expect(clippy::string_slice, ...)]`.
2. For each `&s[..n]` / `&s[n..]` / `&s[a..b]` site that clippy now flags, either:
   - Refactor to `s.floor_char_boundary(n)` / `s.ceil_char_boundary(n)` and slice with the resulting boundary, or
   - Wrap the slice in a narrow `#[expect(clippy::string_slice, reason = "...")]` block whose reason names the specific invariant (e.g. "`pos` came from `s.find('"')` which returns a UTF-8 boundary").
3. Confirm `just clippy-strict` is clean.
