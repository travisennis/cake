# Profile and Optimize Cake Performance

**Source:** Active ExecPlan `performance-improvements.md`

**Priority:** P2

**Effort:** L

**ExecPlan:** `.agents/exec-plans/active/performance-improvements.md`

**Depends on:** None

**Problem:** Cake has no profiling infrastructure or benchmarks. Performance work happens by guess rather than measurement. I/O-bound operations (JSON serialization, session load/save, HTTP overhead) have not been profiled.

**Files involved:** `Cargo.toml` (profile config), `justfile` (profile recipe), hot-path source files as revealed by profiling

**Fix:**
1. Confirm release profile baseline and add a `[profile.profiling]` with debug symbols.
2. Add a `just profile` recipe using `samply` and define 2-3 reproducible workloads.
3. Profile representative workloads, categorize hotspots, and implement only measured source-level improvements.
4. Optionally add Criterion benchmarks for serialization and session loading.
5. Run `just ci` after each change.

Compiler-level experiments (fat LTO, PGO, BOLT) are deferred until after code-level fixes are validated.
