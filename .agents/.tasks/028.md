# Warn (Don't Silently Swallow) StreamRecord Serialization Failures

**Source:** Review of staged v4 session migration changes

**Problem:** In `src/clients/agent.rs`, `stream_record()` serializes `StreamRecord` with `.ok()`:

```rust
let stream_json = self
    .streaming_callback
    .as_ref()
    .and_then(|_| serde_json::to_string(&record).ok());
```

If serialization fails (unlikely with current types, but possible if a new non-serializable field is added), the record is silently dropped from stream-json output. No log, no error, nothing.

**Fix:** Log a warning on failure. The simplest approach:

```rust
.and_then(|_| serde_json::to_string(&record)
    .inspect_err(|e| tracing::warn!("Stream serialization failed: {e}"))
    .ok())
```

Or use a `match` block if inspect_err isn't available on the toolchain.

**Files involved:**
- `src/clients/agent_observer.rs` — `stream_record()` method (the method was moved from `agent.rs` to `agent_observer.rs` in a prior refactor)

**Status:** Completed
