# Document Meaningful Exit Codes

**Source:** product-ideas.md — Exit Codes

**Problem:** Not clear if cake distinguishes success/failure modes for scripting.

**Idea:** Ensure meaningful, documented exit codes:
- `0` — success
- `1` — agent/tool error
- `2` — API error (rate limit, auth failure, network error)
- `3` — input error (no prompt provided, invalid flags)
