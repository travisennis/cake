# Add --output-file Flag with Metadata

**Source:** product-ideas.md — Output Capture

**Problem:** Output goes to stdout but there is no first-class way to save both the answer and session metadata together.

**Idea:** `--output-file result.md` flag. A built-in flag could also write metadata (model, session ID, token usage) as a YAML frontmatter block. `tee` already works, but a native flag is cleaner.
