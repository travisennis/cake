# Fix serde_json to_value Error Policy

**Source:** prompt.md item 27

**Priority:** P2

**Effort:** S

**Problem:** `serde_json::to_value(self).unwrap_or_else(...)` hides serialization failures for a typed value where failure policy should be explicit. After task 061 the offending helper is `ConversationItem::to_api_input()`, which is now `#[cfg(test)]`-only and wraps `serde_json::to_value(self.to_api_input_item())` in a fallback that emits a `{"type":"error", ...}` JSON object. The fallback can never trigger for the current DTO shapes and would silently mask a real serde bug if it ever did.

**Files involved:** `src/clients/types.rs`

**Fix:** Either delete `to_api_input()` and inline `serde_json::to_value(item.to_api_input_item()).unwrap()` at the test call sites, or replace the `unwrap_or_else` body with an `expect` that documents why the conversion is infallible.
