# Stop Treating Literal TMPDIR as Safe rm Target

**Source:** prompt.md item 35

**Priority:** P0

**Effort:** S

**Problem:** `check_dangerous_rm` accepts `$TMPDIR` and `${TMPDIR}` text as safe without knowing what the shell will expand them to. If the variable is empty or unexpected, the actual delete target can be unsafe.

**Files involved:** `src/clients/tools/bash_safety.rs`

**Fix:** Reject shell variable based `rm -rf` targets or resolve environment variables before classification with strict fallback behavior. Add tests for unset and empty TMPDIR patterns.

**Status:** Completed

**Resolution:** Removed `$TMPDIR` and `${TMPDIR}` from the `rm -rf` temporary-directory allowlist so only literal `/tmp` and `/var/tmp` targets are treated as safe. Added regression coverage for direct and multi-target shell-variable forms.

**Validation:**
- `cargo test bash_safety`
- `cargo fmt --check`
- `just ci`
