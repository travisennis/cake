# Implement Lazy Skill Body Loading at Activation Time

**Status:** Pending
**Priority:** P1
**Effort:** L
**Created:** 2026-05-14
**ExecPlan:** Required before implementation
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

`Skill::load_body` exists with a docstring claiming "lazy-loaded at activation time, not during discovery", but production code never calls it. Commit `3cc2e73` masked this by gating it (and `SkillCatalog::get_skill_by_location`) behind `#[cfg(test)]`, which makes the API genuinely unavailable in production while the misleading docstring remains.

Skill setups can grow large — both in number of skills and per-skill body size. Lazy loading at activation is a real win for memory and startup: discovery scans frontmatter only, body bytes are read on demand when a skill is activated.

## Files Involved

- `src/config/skills.rs` (`Skill`, `SkillCatalog`, frontmatter parser, `load_body`, `get_skill_by_location`)
- `src/clients/agent.rs` (`SkillActivations`, activation flow)
- Anywhere skill body content is currently injected into prompts (search for `skill.body` / `skill.content` / equivalent)

## Fix Direction

The implementation needs an ExecPlan in `.agents/exec-plans/active/` per `.agents/PLANS.md`. Outline:

1. Confirm current loading behavior: when (and where) is the full body of a skill currently read into memory? Discovery, or already lazy?
2. If discovery currently slurps body bytes, change discovery to read frontmatter only and store `location: PathBuf` plus parsed metadata.
3. Wire `Skill::load_body` (or a renamed equivalent) into the activation path so body content is read the first time the skill activates in a session.
4. Decide on caching: per-session memoization, process-wide cache, or no cache (re-read each activation). Preference: per-session memoization keyed by skill location.
5. Remove `#[cfg(test)]` gating from `load_body` and `get_skill_by_location`; both should be production APIs.
6. Update the docstring on `load_body` to match reality and document the caching policy.
7. Add tests covering: discovery-without-body (assert no read of body bytes), activation-loads-body, caching behavior.
