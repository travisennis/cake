# Replace Boolean Task Completion API with Outcome Enum

**Source:** prompt.md item 8

**Priority:** P1

**Effort:** M

**Problem:** Task completion accepts `success: bool`, `result_text: Option<String>`, and `error_message: Option<String>`, allowing invalid state combinations.

**Files involved:** `src/clients/agent.rs`, `src/clients/types.rs`

**Fix:** Introduce `TaskOutcome` with success and failure variants. Update call sites and serialization so impossible states are unrepresentable internally.

**Status:** Completed
