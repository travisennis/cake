# Fix Session Save Requiring Valid UUID at Runtime

**Source:** fix.md #3

**Problem:** `DataDir::save_session()` validates `session.id` as a UUID at runtime via `uuid::Uuid::parse_str`. Non-UUID IDs cause a runtime error with no compile-time enforcement.

**Files involved:**
- `src/config/data_dir.rs` (lines 124–139)
- Any code that constructs `Session`

**Fix:** Enforce the UUID invariant at construction time. Modify `Session::new` to accept a `uuid::Uuid` instead of `String`, or add `TryFrom<String>`. Store internally as `uuid::Uuid` and convert only for serialization. Update all callers.

**Status:** Completed
