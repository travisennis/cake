# Fix mtime-Based Newest Session Resolution Fragility

_This task was Blocked with no documented blocker. The fix direction is clear (option 1: add created_at timestamp field). Moving to Pending._

**Source:** fix.md #5

**Problem:** `load_latest_session()` picks the session file with the highest `modified` timestamp via `max_by_key`. This is fragile due to `touch`, clock skew, and backup/restore operations.

**Files involved:**
- `src/config/data_dir.rs` (lines 164–198)

**Fix:** Stop relying on filesystem mtime. Options:
1. Add a `created_at: DateTime<Utc>` field to `Session`, persist it, and use it as the sort key.
2. Maintain a small `latest` pointer file updated atomically in `save_session()`.

Option 1 is preferred.

**Status:** Blocked
