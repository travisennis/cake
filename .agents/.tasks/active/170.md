# Move Chat Response Parsing Tests into Extracted Test Module

**Created:** 2026-05-23
**Source:** `review.md` Issue 2

## Problem

`src/clients/chat_completions.rs` uses two test organization patterns at once:

- `#[path = "chat_completions_tests.rs"] mod tests;`
- An inline `response_parsing_tests` module of roughly 380 lines

The current split leaves production code clutter in a file that was already partially converted to extracted tests, and future contributors have to decide which pattern to follow.

## Relevant Files

- `src/clients/chat_completions.rs`
- `src/clients/chat_completions_tests.rs`

## Fix Direction

Move the inline `response_parsing_tests` module into `chat_completions_tests.rs`, keeping the test names and assertions unchanged. If needed, add a short comment in the extracted test file explaining that it includes `wiremock`-backed response parsing tests.

Do not change production behavior.

## Acceptance Notes

- `chat_completions.rs` contains only the external `#[path = "chat_completions_tests.rs"] mod tests;` test hook.
- The response parsing tests still compile and run from `chat_completions_tests.rs`.
- Targeted `cargo test chat_completions` passes.
- `cargo fmt` and `just ci` pass before handoff.
