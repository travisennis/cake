# Add Per-Session Telemetry Sidecar

**Source:** Active ExecPlan `per-session-telemetry-plan.md`

**Priority:** P2

**Effort:** XL

**ExecPlan:** `.agents/exec-plans/active/per-session-telemetry-plan.md`

**Depends on:** None

**Problem:** Cake has no session-scoped performance timeline. The daily log mixes unrelated runs, and the session transcript captures conversation state but not request timings, retry reasons, tool durations, or invocation metadata. Debugging slow sessions requires reconstructing timing from logs.

**Files involved:** `src/session_telemetry.rs` (new), `src/config/data_dir.rs`, `src/clients/agent.rs`, `src/main.rs`, `tests/session_telemetry.rs` (new), `docs/design-docs/session-management.md`, `docs/design-docs/logging.md`, `README.md`, `.agents/skills/debugging-cake/SKILL.md`

**Fix:**
1. Define telemetry record schema and an append-only NDJSON writer for `{session_id}.ndjson` under `~/.cache/cake/session-telemetry/`.
2. Wire telemetry into session startup/resume/fork detection and final session summary in `main.rs`. Skip entirely when `--no-session` is set.
3. Instrument `Agent` to record API attempt timing, retry decisions, and tool call durations.
4. Add integration tests proving sidecar creation, retry recording, and `--no-session` behavior. Update docs so the sidecar is discoverable.

Telemetry is best-effort: write failures are logged once, then telemetry is disabled for the rest of that invocation. The session transcript remains the authoritative record.
