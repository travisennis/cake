# Ensure Worktree Cleanup Runs After Early Failures

**Created:** 2026-05-24
**Source:** `.clawpatch/findings/fnd_sig-feat-cli-command-c0e2bf8f9b-_fa3e8f6725.json`
**Finding:** `fnd_sig-feat-cli-command-c0e2bf8f9b-_fa3e8f6725`

## Problem

`setup_worktree` creates and enters the temporary git worktree before stdin/content validation. If stdin reading or `build_content` fails, `prepare_run` returns an error without producing a `PreparedRun`, so the cleanup block in `CodingAssistant::run` cannot execute.

Even after `prepare_run` succeeds, fallible operations that return via `?` before the explicit cleanup block can leave the extra worktree registered.

**Confirmed by code review (2026-06-09):** every `?` between `prepare_run` and the cleanup block at the end of `CodingAssistant::run` skips cleanup — that includes `load_run_resources`, `build_client_and_session`, `prepare_seeded_session`, the `attach_*` helpers, `execute_agent_turn` (hook errors propagate even though agent errors are captured in `TurnResult`), and `render_turn`. `worktree::Worktree` has no `Drop` impl, so nothing backstops these paths. A `Drop`-based guard is the most robust fix because it would also cover the Ctrl-C interrupt path (see task 200, which depends on cleanup running on interrupt).

## Relevant Files

- `src/main.rs`
- `tests/stdin_handling.rs`
- Integration coverage for `--worktree` failure paths

## Reproduction

From a git repository, run:

```bash
cake --worktree -
```

without piping stdin. The command errors with `No input provided via stdin`, but the worktree created by `setup_worktree` can remain registered because cleanup is skipped on that exit path.

## Fix Direction

Ensure worktree cleanup runs on every exit path after worktree creation. Viable approaches include:

- Validate stdin/content before creating the worktree where possible.
- Wrap the post-setup run in a result-capturing block and perform cleanup in a finally-style section.
- Introduce a guard type whose `Drop` restores the original directory and removes unchanged worktrees.

## Acceptance Notes

- `cake --worktree -` with no stdin returns the expected input error and leaves no extra git worktree registered.
- Fallible paths after worktree setup still clean up unchanged temporary worktrees.
- Cleanup restores the original directory before removing the worktree.
- A regression test covers the no-stdin `--worktree` path.
- `cargo check --tests`, `cargo fmt`, and `just ci` pass.
