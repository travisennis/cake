# Remove No-Op `_ = result.as_bytes()` In bash.rs Test

**Status:** Pending
**Priority:** P3
**Effort:** XS
**Created:** 2026-05-14
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

The test in `src/clients/tools/bash.rs` (around line 539-542) contains:

```rust
// Verify the result is valid UTF-8 (would panic if not)
_ = result.as_bytes();
```

Both the comment and the line are misleading:

- `result` is a `String`, which by construction holds valid UTF-8.
- `as_bytes()` returns `&[u8]` and never panics.
- The expression has no side effect; the `_ =` discard makes it a no-op.

The truncation under test uses `floor_char_boundary`, so UTF-8 validity is structurally guaranteed; an `assert!` would not add real coverage.

## Files Involved

- `src/clients/tools/bash.rs` (around line 539-542)

## Fix Direction

Delete the line and its comment.
