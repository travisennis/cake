# Reassess Linux Landlock Default Build Policy

**Created:** 2026-05-24
**Source:** `.clawpatch/findings/fnd_sig-feat-config-7afc533f22-d9530_8eae199309.json`
**Finding:** `fnd_sig-feat-config-7afc533f22_d95305e476`
**Related:** Task 100, ADR 006 (`docs/adr/006-linux-landlock-release-artifacts.md`)

## Problem

The finding reports that Linux default builds can omit the Landlock backend because `landlock` is optional and `default = []` in `Cargo.toml`. A normal Linux source build or `cargo install` therefore compiles without Landlock unless the caller passes `--features landlock`.

This is the same underlying risk addressed by task 100, but the finding proposes a different fix than the accepted task 100 decision.

## Task 100 Comparison

Task 100 decided that official Linux release artifacts must be built with `--features landlock`, while Cargo's global default feature set remains empty. ADR 006 explicitly rejected enabling `landlock` in Cargo default features because Cargo defaults are global and Landlock is Linux-only.

The finding is not a different issue from task 100. It is a challenge to task 100's accepted policy: it recommends making Landlock part of the default Linux build, either through default features or a target-specific non-optional dependency.

## Relevant Files

- `Cargo.toml`
- `.github/workflows/release.yml`
- Linux CI configuration
- `docs/adr/006-linux-landlock-release-artifacts.md`
- README and sandbox documentation

## Fix Direction

Triage whether task 100's accepted policy is still sufficient, or whether the project should change the policy so Linux source/default builds include Landlock without requiring an explicit feature flag.

If the policy changes, update ADR 006 or supersede it with a new ADR before implementation. Then update Cargo feature/dependency configuration, CI checks, and documentation accordingly.

If the task 100 policy stands, close this ticket with a note that official artifacts are covered by ADR 006 and source builds intentionally remain explicit opt-in.

## Acceptance Notes

- The project has an explicit decision on whether Linux source/default builds must include Landlock.
- ADR 006 is confirmed, amended, or superseded to match the decision.
- If the policy changes, CI verifies the chosen default Linux build behavior.
- Documentation clearly distinguishes official release artifact behavior from source build or downstream packager behavior.
