# Add Settings-Based and Profile-Based System Prompt Configuration

**Status:** Open
**Priority:** P3
**Effort:** M
**Created:** 2026-05-17
**Source:** Follow-up from task 150 design discussion
**ExecPlan:** -
**Depends on:** 150 (Move Built-In System Prompt to Markdown With Runtime Overrides)

## Problem

Task 150 introduced file-based system prompt overrides via `.cake/system.md` and `~/.config/cake/system.md`. However, there is no way to configure the system prompt through `settings.toml`, which means:

- No way to specify a custom system prompt path in settings
- No way to use different system prompts per profile (e.g., a different prompt for a "review" profile vs. a "fast" profile)
- No `--system-prompt` CLI flag for one-off prompt overrides

## Proposed Direction

Add a `system_prompt` key to `settings.toml` that accepts a file path:

```toml
system_prompt = "/path/to/my-prompt.md"
```

And extend profiles to support per-profile system prompts:

```toml
[profiles.review]
default_model = "claude"
system_prompt = "/path/to/review-prompt.md"
```

Also add a `--system-prompt <path>` CLI flag that overrides all other sources.

The precedence chain would become (highest to lowest):
1. `--system-prompt` CLI flag
2. `.cake/system.md` (project-level file)
3. `system_prompt` in project `settings.toml`
4. `~/.config/cake/system.md` (user-level file)
5. `system_prompt` in global `settings.toml`
6. Built-in default

This task requires careful design of the precedence rules and how profile overrides interact with the file-based chain.

## Notes

- This task was explicitly deferred from task 150 to keep scope manageable.
- The file-based override mechanism from task 150 should remain as-is; this task adds settings.toml and CLI integration on top.
