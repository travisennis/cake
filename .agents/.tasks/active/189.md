# Document Lenient Edit Summary Argument Parsing

**Created:** 2026-05-29
**Source:** Review of commit `b4e765140df9`

## Problem

`src/clients/tools/edit.rs` parses `mutating_target` with `EditSummaryArgs`, which intentionally does not use `#[serde(deny_unknown_fields)]`. Full edit execution parses with `EditArgs`, which does deny unknown fields.

The behavior is appropriate because target extraction for duplicate-mutation guarding should be advisory and lenient, while execution should retain strict validation. The asymmetry is not documented near the summary type, so future readers may mistakenly "fix" it by making summary parsing strict.

## Relevant Files

- `src/clients/tools/edit.rs`

## Fix Direction

Add a short comment or doc comment above `EditSummaryArgs` explaining that it is a path-only, lenient view for summaries and duplicate-mutation target extraction. Note that full execution still uses `EditArgs` for strict validation.

## Acceptance Notes

- The parse strictness asymmetry is documented near `EditSummaryArgs`.
- Runtime behavior is unchanged.
- Run `cargo fmt` and the required final verification for code-comment changes if any Rust file is edited.
