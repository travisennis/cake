# Fix `SandboxConfig` `struct_field_names` Reason and Verify Dropped `dead_code`

**Status:** Pending
**Priority:** P2
**Effort:** S
**Created:** 2026-05-14
**Source:** Code review of commit `3cc2e73` (build(config): apply stricter clippy configuration and fix warnings)

## Summary

Commit `3cc2e73` converted `#[allow(clippy::struct_field_names, dead_code)]` on `SandboxConfig` into `#[expect(clippy::struct_field_names, reason = "field names mirror the struct name for clarity")]`. Two problems:

1. **Wrong reason**: the lint fires because of the shared `read_*` prefix (`read_write`, `read_only`), not because field names mirror the struct name (the struct is `SandboxConfig`).
2. **Dropped `dead_code`**: the original `#[allow(... dead_code)]` was load-bearing on platforms where these fields are not consumed by any cfg branch. Need to verify whether non-macOS, non-Linux builds (the `#[cfg(not(any(target_os = "macos", target_os = "linux")))]` branch) still compile.

## Files Involved

- `src/clients/tools/sandbox/mod.rs` (around line 36-44)

## Fix Direction

1. Audit field usage across all platform cfg branches in `src/clients/tools/sandbox/`. Check macOS, Linux + landlock, Linux without landlock, and the unsupported-platform fallback.
2. If `dead_code` is needed on some platform, add a properly cfg-gated `#[expect(dead_code, reason = "...")]` for that platform only, or restructure so all fields are consumed (e.g., explicit `let _ = &self.read_write;` in the unsupported branch).
3. Either:
   - **Rename the fields** to remove the shared prefix (e.g., `writable: Vec<PathBuf>`, `readable: Vec<PathBuf>`, plus rename `work_dir` if it falls under the same lint), and remove the `#[expect(clippy::struct_field_names)]` entirely, OR
   - **Keep the names** and rewrite the reason: `reason = "fields share read_* prefix to group filesystem permissions; renaming would obscure intent"`.
4. Confirm `just clippy-strict` and `just ci` are clean on macOS. Cross-compile or document Linux verification.

## Acceptance Notes

- Reason text matches the actual lint trigger.
- `dead_code` allowance is restored on platforms that need it, or fields are restructured so no platform produces the warning.
- `just ci` passes on macOS.
- Linux verification noted in the task or executed if a Linux runner is available.
