# Serialize Mutating Tool Calls Within a Turn

**Created:** 2026-06-09
**Source:** Code review of `src/clients/agent/agent_loop.rs` (2026-06-09)

## Problem

All tool calls in an assistant turn execute concurrently via
`futures::future::join_all` (`src/clients/agent/agent_loop.rs:216`). The
duplicate-mutation guard from task 163 only rejects a second Edit/Write
targeting the same canonical path. It does not cover:

- Two Bash calls in one turn (e.g., `mkdir foo` + `mv x foo/`, or
  `cargo build` + `cargo test`) — these race with each other.
- A Bash call and an Edit/Write touching the same file — the guard cannot
  know what paths a shell command mutates.

Models routinely emit order-dependent tool calls in a single turn and assume
sequential semantics. Concurrent execution makes the interleaving
nondeterministic and can corrupt multi-step shell workflows. Comparable agent
CLIs execute tool calls sequentially for this reason.

## Relevant Files

- `src/clients/agent/agent_loop.rs` — tool execution futures and `join_all`
- `src/clients/tools/duplicate_guard.rs` — existing same-path guard
- Task 163 (completed) and task 164 (completed prompt guidance) for prior art
- `docs/design-docs/` — tool execution semantics are a documented
  compatibility surface

## Fix Direction

- Execute mutating tools (Bash, Edit, Write) sequentially in the order the
  model emitted them; keep Read calls concurrent (or run everything
  sequentially if the simplicity is worth more than the latency win).
- Keep the duplicate-mutation guard: it still gives the model a clear error
  for redundant same-file mutations.
- Tool execution semantics are a compatibility surface per AGENTS.md, so this
  needs an ADR before implementation.

## Acceptance Notes

- Two Bash calls in one turn observably execute in emission order (test via
  stub backend and commands with ordering-sensitive effects).
- Read-only calls still run concurrently if that design is kept.
- Telemetry per-tool timing still records correctly under serialization.
- ADR recorded for the execution-ordering decision.
- `just ci` passes.
