# Prevent Concurrent Same-File Mutating Tool Calls

**Status:** Pending
**Priority:** P1

## Problem

Cake currently lets the model issue multiple mutating tool calls for the same file in one assistant turn. In session `a91aa043-4833-4118-88ff-92feef8688b9`, this appears to have caused lost edits and trailing file corruption while task 099 was rewriting doctests.

Evidence:

- Lines 69-70: two `Edit` calls targeted `src/config/model.rs` in the same model turn.
- Lines 76-81: six `Edit` calls targeted `src/config/settings.rs` in the same model turn.
- Lines 104-109: six `Edit` calls targeted `src/config/data_dir.rs` in the same model turn.
- Line 134: `cargo check --tests` failed with `unexpected closing delimiter` in `src/config/model.rs`.
- Lines 170-179: the agent found and removed trailing garbage in `src/config/model.rs`.
- Lines 361-387: the same corruption pattern appeared in `src/config/settings.rs`.

The task eventually succeeded, but recovery consumed many turns and tool calls.

## Relevant Files

- `src/clients/agent.rs`
- `src/clients/tools/mod.rs`
- `src/clients/tools/edit.rs`
- `src/clients/tools/write.rs`
- Conversation/tool execution tests around parallel tool calls

## Fix Direction

Make mutating tool execution path-aware. Possible approaches:

- Serialize `Edit` and `Write` calls that target the same path within one assistant turn.
- Reject duplicate same-path mutating calls in the same turn with a clear tool result telling the model to re-read and edit sequentially.
- Keep parallel execution for reads and independent writes to different paths.

The implementation should preserve deterministic call/result ordering so persisted sessions remain understandable.

Because this changes agent-loop and tool-execution behavior, create an ExecPlan before implementation.

## Acceptance Notes

- Multiple `Edit` or `Write` calls targeting the same file in one turn cannot race or corrupt the file.
- Parallel reads and independent writes to different files still work.
- Tool results explain any rejected same-path mutating call in actionable language.
- Tests cover same-file `Edit`/`Edit`, `Edit`/`Write`, different-file mutations, and non-mutating parallel calls.
- `cargo check --tests`, `cargo fmt`, and `just ci` pass.
