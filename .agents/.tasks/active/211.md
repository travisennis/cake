# Apply Small Agent Module Polish From Code Review

**Created:** 2026-06-09
**Source:** Code review of `src/clients/agent.rs` (2026-06-09)

## Problem

Three small cleanups in the agent module, bundled because each is too small
to track alone:

1. The mutex-poison-recovery closure
   (`lock().unwrap_or_else(|e| { tracing::error!(...); e.into_inner() })`)
   is duplicated at `src/clients/agent.rs:218`, `src/clients/agent.rs:242`,
   and in `src/clients/skill_dedup.rs`. Extract a small helper (e.g.,
   `lock_recovering(&Mutex<T>) -> MutexGuard<T>`).
2. `with_activated_skills` (`src/clients/agent.rs:216`) takes `self` and
   mutates through the mutex inside a block, while every sibling builder
   takes `mut self`. Align the style.
3. `emit_prompt_context_records` (`src/clients/agent.rs:338`) collects
   records into a `Vec` and then iterates it by reference; it can persist
   directly in a single pass without the intermediate collection.

## Relevant Files

- `src/clients/agent.rs`
- `src/clients/skill_dedup.rs`

## Fix Direction

Mechanical refactors, no behavior change. Keep each change surgical and
covered by the existing tests.
