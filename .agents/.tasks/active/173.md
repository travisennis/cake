# Document Module Size Linter Parsing Limitations

**Created:** 2026-05-23
**Source:** `review.md` Issue 7

## Problem

`scripts/lint-module-size.py` explicitly uses brace counting and a `"test" in module name` heuristic. The script is informational and always exits 0, which is acceptable for the current use.

The script header documents the broad approach but does not spell out known edge cases. If this lint is later promoted to a gate, contributors may overestimate the accuracy of the current parser.

## Relevant Files

- `scripts/lint-module-size.py`

## Fix Direction

Add concise documentation in the script header or comments covering known limitations:

- Module names such as `contest` or `protest` can be classified as tests.
- Braces inside strings or comments can affect brace-depth counting.
- Nested test modules may be counted imperfectly.
- A parser-backed implementation should be considered before making the lint blocking.

Keep the script behavior unchanged unless a small test fixture exists and needs a documentation-only update.

## Acceptance Notes

- Known parser limitations are documented near the existing script overview.
- The script remains informational and exits 0.
- `just lint-module-size` still runs.
