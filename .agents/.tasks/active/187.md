# Extract Immediate Tool Error Result Helper

**Created:** 2026-05-29
**Source:** Review of commit `b4e765140df9`

## Problem

`Agent::send` now has duplicated synchronous-error result construction in the `ScheduledToolPlan::RejectedDuplicateMutation` and `ToolHookPlan::Block` execution arms. Both paths build a `ToolRunResult` with error telemetry, no skill activation, and effectively immediate duration; only the output string differs.

The duplication obscures the important difference between duplicate-mutation rejection and hook blocking.

## Relevant Files

- `src/clients/agent.rs`

## Fix Direction

Extract a small helper for immediate tool error results, for example:

```rust
fn immediate_tool_error_result(
    name: &str,
    call_id: &str,
    output: String,
    turn_index: u32,
) -> ToolRunResult
```

Use it from both the duplicate-mutation rejection arm and the hook-block arm. Preserve current telemetry semantics, including `was_error: true`, `output_bytes`, `call_id`, and `skill_activation: None`.

## Acceptance Notes

- Duplicate `ToolRunResult` construction is removed from the two synchronous error arms.
- The hook-block and duplicate-mutation outputs remain unchanged.
- Run a focused agent check such as `cargo test duplicate_guard`, then `cargo fmt`, then the required final verification for code changes.
