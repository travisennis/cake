# Align Self-Reflection Note Instructions With Tool Write Permissions

**Created:** 2026-05-18
**Source:** Evaluation of session `a77cc195-424e-4d97-b8e8-1788f5ffb39f`

## Problem

The default system prompt instructs agents to append mistakes, desires, and
learnings to files under `~/.cake/`, but the normal tool write contract does not
clearly make that path writable. The Write/Edit tools allow writes in the
working directory, temp directories, and configured settings directories. Bash
sandboxing also primarily grants write access to the cwd, temp directories, and
configured settings directories.

In the evaluated session, the agent explicitly recognized a mistake after
using `git add -A` and briefly committing two unrelated untracked files, but it
did not append anything to `~/.cake/MISTAKES.md`.

This makes the self-reflection instruction unreliable: either the agent ignores
it, or following it may require a path that the tools reject unless the user has
configured `~/.cake` as writable.

## Evidence

- Session line 103, call `call_00_hSFnofbniFEljH0v1s6U3581`: `git add -A`
  created commit `14f6ade` and included unrelated files:

```text
create mode 100644 prompt.md
create mode 100644 workflow-management.md
```

- Session line 110 reasoning: the agent noticed the mistake:

```text
I accidentally included two unrelated untracked files ... because I used git add -A.
```

- Session lines 118-143: the agent reset and recommitted with only the intended
  files.
- Session records contain no Read/Edit/Write/Bash call that appends to
  `~/.cake/MISTAKES.md`.
- `src/prompts/system.md` instructs agents to append mistakes to
  `~/.cake/MISTAKES.md`, desires to `~/.cake/DESIRES.md`, and learnings to
  `~/.cake/LEARNINGS.md`.
- `src/clients/tools/write.rs` and `src/clients/tools/mod.rs` restrict writes
  to the cwd, temp directories, and configured settings directories.

## Relevant Files

- Settings docs for configured writable directories

## Fix Direction

Choose one durable contract and make the prompt, tools, docs, and tests agree.

Candidate approaches:

- Move self-reflection notes into a writable project-local path, such as
  `.cake/agent-notes/`, and update prompt snapshots.
- Treat `~/.cake` as an explicit configured settings directory when it exists,
  and document that this path is writable for self-reflection notes.
- Remove the self-reflection instruction from the default prompt if cake does
  not want agents writing persistent user-level notes.

The fix should avoid telling the model to perform a write that is rejected by
the default tool permissions.

## Acceptance Notes

- The default prompt no longer asks for writes to paths that are not writable by
  default.
- If self-reflection notes remain, tests or snapshots cover the updated prompt.
- Tool path validation and sandbox behavior are documented consistently with the
  prompt.
- A session where the agent recognizes a mistake has a feasible, documented way
  to record it, or the prompt no longer requests that behavior.
- `just ci` passes if code or prompt snapshots change.
