# Include Problematic Input in Edit Tool Parse Error Messages

**Source:** Session analysis of `05f274d2-f14e-4e7f-8bc2-aa6074b2831d` — the model sent a
malformed Edit arguments JSON and received:
```
Error: Invalid edit arguments: trailing characters at line 1 column 231
Expected shape: {"path":"file.txt","edits":[{"old_text":"exact text to replace","new_text":"replacement text"}]}
```

The error does not show what the model sent, making it hard for the model
(or a human debugging) to understand which part of the input was wrong.
The model had to reverse-engineer the mistake from the column number alone.

**Problem:** `parse_edit_args` in `src/clients/tools/edit.rs` delegates to
`serde_json::from_str` and formats only serde's error message plus the
expected shape. The actual arguments text is not included.

**Fix:** Include a preview (e.g. first 200 characters) of the raw arguments
in the error message so the model can see its own malformed input.

**Files involved:** `src/clients/tools/edit.rs`

**Acceptance:**
- When Edit arguments fail to parse, the error message includes a preview of
  the input (truncated to avoid flooding context).
- All existing tests pass.
- `cargo fmt` passes.
