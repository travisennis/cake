# Add OSC 9;4 Terminal Progress Indicator

**Source:** Active ExecPlan `progress-bar-plan.md`

**Priority:** P3

**Effort:** S

**ExecPlan:** `.agents/exec-plans/active/progress-bar-plan.md`

**Depends on:** None

**Problem:** Cake has an `indicatif` text spinner in the terminal content area but does not set a progress indicator on the terminal tab or taskbar. In terminals that support OSC 9;4 (iTerm2, Windows Terminal, WezTerm), there is no visual cue that cake is working when the terminal window is not focused.

**Files involved:** `src/progress.rs` (new), `src/main.rs`

**Fix:**
1. Create `src/progress.rs` with OSC 9;4 sequence formatting and terminal-gated write functions. Only emit sequences when stdout is a real terminal (not piped/redirected).
2. Wire `start_progress()` and `stop_progress()` into the run loop in `main.rs` alongside the existing spinner lifecycle. Use a `ProgressGuard` Drop type so the indicator is always cleared on error.
3. Add unit tests for sequence formatting.

Only states 0 (off) and 3 (indeterminate) are needed. No changes to `Cargo.toml` — the `is_terminal()` check is stable since Rust 1.70.

**Status:** Pending
