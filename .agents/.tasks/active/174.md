# Remove Directory Reading Support from the Read Tool

## Summary

The `Read` tool currently handles both files (returning line-numbered content) and directories (returning a flat listing). Remove the directory branch so `Read` only handles files. Agents can use `Bash` with `ls`, `find`, or `tree` for directory listings.

## Relevant Files

| File | What needs changing |
|---|---|
| `src/clients/tools/read.rs` | Remove `read_directory()` function and `is_dir()` branch in `execute_read()`. Remove directory-specific tests (`read_directory_listing`, `read_empty_directory`). |
| `src/clients/tools/read-description.txt` | Remove mentions of directory listing and "replaces ls". Describe file-only behavior. |
| `docs/design-docs/tools.md` | Update Read tool description, purpose, and parameter docs to remove directory listing. Update line ~40 (example), ~246 (purpose), ~250 (parameters), ~257 (output examples). |

## Implementation Notes

1. In `read.rs`, replace the `if path.is_dir()` branch with an error return like `"Path is a directory, not a file: {}"`.
2. Keep `read_file()` and all its associated logic (binary detection, line ranges, truncation) unchanged.
3. Keep `extract_path()` and `summarize_args()` — they are used by the skill activation path and session recording and don't need modification (they parse args, not paths).
4. Update `read-description.txt` to describe file-only reading. Remove "Use this instead of cat/head/tail/ls via Bash."
5. Update `docs/design-docs/tools.md` references (lines ~40, ~246, ~250, ~257, and possibly ~392).
6. Delete or update the directory-specific tests; keep all file-specific tests.
7. Ensure existing tests still pass (no structural changes to file reading logic).
8. Run `cargo fmt` and `just ci` (or at minimum `cargo test tools::read` and `cargo check --tests`).

## Considerations (from code review)

- **Sandbox bypass symmetry**: Read reads files via `std::fs` (bypassing sandbox). Directory listing also bypasses sandbox. Removing directory support aligns with the principle that file I/O goes through Read but filesystem exploration goes through Bash (which is sandboxed). This is a *pro-removal* argument.
- **Session backward compat**: Saved sessions with recorded directory reads replay fine — results are already stored in session files.
- **Skill activation**: `extract_path()` intercepts Read calls for skill paths (always `SKILL.md` files). Not affected.
- **Tool parameter schema**: Stays the same (`path`, `start_line`, `end_line`). Calling `Read` on a dir will produce a clear error instead of a listing.
- **No integration tests** reference directory reading with the Read tool.

## Acceptance Notes

- `Read` with a directory path returns a clear error message and no listing.
- `Read` with a file path works exactly as before (binary detection, line ranges, truncation, window behavior).
- `cargo test tools::read` passes (directory tests removed or updated; file tests pass).
- `cargo fmt` and `cargo check --tests` pass.
- Tool description no longer mentions directory listing or `ls`.
