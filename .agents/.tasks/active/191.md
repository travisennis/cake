# Remove `ahm` Dependency From CI Task Index Check

**Created:** 2026-05-29  
**Source:** Evaluation of session `d8ffbb40-0710-4172-8580-e9e36b35c066`

## Problem

`just ci` currently depends on `ahm --dry-run index` through the
`task-index-check` recipe:

```makefile
task-index-check:
    @OUT="$(ahm --dry-run index)"; \
    if [ -n "$OUT" ]; then \
        printf '%s\n' "Task indexes are stale. Regenerate with \`ahm index\`."; \
        printf '%s\n' "$OUT"; \
        exit 1; \
    fi
```

This is the wrong dependency boundary for CI. `ahm` is a local workflow tool and
is not yet available as an installable GitHub Actions dependency. Even if the
current `ahm --dry-run index` behavior is fixed in `ahm`, cake's CI should not
require a separate unpublished app to validate repository consistency.

There is also a current local behavior issue: `ahm --dry-run index` prints all
7 generated index file paths even when the files are already current, so the
existing recipe can fail locally after a fresh `ahm index` or
`ahm index --force`. That behavior should be fixed in `ahm`, not papered over
as the primary cake-side fix.

This has caused significant agent confusion in at least two sessions:

- Session `d8ffbb40` (this session): the agent spent ~9 turns and 10+ commands
  investigating whether the failure was caused by its changes or was pre-existing,
  including running `ahm --help`, `ahm version`, `ahm index --force`, `which ahm`,
  `git stash` tests, and inspecting `fd`/`ls` output.

- Task 163 acceptance notes (line 65) document the same blocker: `just ci` was
  attempted but blocked at `task-index-check` because `ahm --dry-run index`
  still reports all generated indexes stale immediately after `ahm index` and
  `ahm --force index`.

## Acceptance

- `just ci` no longer requires `ahm` to be installed.
- GitHub Actions can run the task index consistency check using only tools that
  are already available in CI or installed by the repository's CI setup.
- The local workflow still has a convenient way to regenerate indexes with
  `ahm index`.
- The replacement check exits 0 when generated indexes are current.
- The replacement check exits non-zero when a task, research note, or ExecPlan
  change should have regenerated indexes but did not.
- The task notes distinguish the cake-side CI dependency fix from the separate
  upstream `ahm --dry-run index` behavior fix.

## Relevant Files
