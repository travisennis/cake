# Exclude extracted test modules from cargo-crap

## Problem

`scripts/cargo-crap.sh` maintains the extracted Rust test module exclusions by hand, listing each `*_tests.rs` file individually. That list is already stale: `src/clients/tools/edit_tests.rs` exists and is an extracted test file, but it was not added to the exclusions.

Because extracted test files have no matching LCOV entries, complex helpers in a newly extracted test module can silently spike CRAP and re-trip the gate until someone notices and patches the list.

## Relevant Files

- `scripts/cargo-crap.sh` - current hand-maintained exclude list for cargo-crap.
- `src/clients/tools/edit_tests.rs` - observed extracted test file missing from the exclude list.
- Other extracted `*_tests.rs` files under `src/`.

## Fix Direction

Replace the per-file extracted test module entries with a glob such as `--exclude '**/*_tests.rs'`, mirroring the existing `tests/**` exclusion. Confirm the glob syntax works with the cargo-crap wrapper and still excludes all extracted test modules.

## Acceptance Notes

- [ ] `scripts/cargo-crap.sh` no longer requires one exclusion entry per extracted `*_tests.rs` file.
- [ ] `src/clients/tools/edit_tests.rs` and future `**/*_tests.rs` files are excluded from CRAP scoring.
- [ ] The cargo-crap coverage check still analyzes production source files and still excludes integration tests under `tests/**`.
- [ ] Verification includes the relevant cargo-crap/check-coverage command or a documented dry-run equivalent.
