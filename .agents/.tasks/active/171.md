# Extract Inline Tests from Remaining Large Modules

**Created:** 2026-05-23
**Source:** `review.md` Issue 4

## Problem

Several large modules still combine production code with large inline `#[cfg(test)]` modules. The recent extracted-test work established a repeatable pattern, but large files remain harder to scan than necessary.

Current line counts from `wc -l`:

| File | Lines |
| ---- | ----- |
| `src/clients/agent.rs` | 2413 |
| `src/main.rs` | 1697 |
| `src/clients/responses.rs` | 1664 |
| `src/hooks.rs` | 1449 |
| `src/types/session.rs` | 1109 |
| `src/config/skills.rs` | 1065 |
| `src/clients/tools/bash.rs` | 1016 |

The review claimed all seven are flagged by `just lint-module-size`; the actual current linter output flags only a subset based on production-line and test-ratio thresholds. The underlying consistency debt is still valid: large inline test blocks should be extracted where doing so reduces production-file clutter without changing behavior.

## Relevant Files

- `src/clients/agent.rs`
- `src/main.rs`
- `src/clients/responses.rs`
- `src/hooks.rs`
- `src/types/session.rs`
- `src/config/skills.rs`
- `src/clients/tools/bash.rs`
- Existing extracted test modules for examples, such as `src/clients/chat_completions_tests.rs`

## Fix Direction

Create an ExecPlan before implementation because this is broad, multi-module refactoring work.

Extract inline tests into sibling `*_tests.rs` modules using the established `#[cfg(test)] #[path = "..."] mod tests;` pattern. Work in small batches and prefer zero production behavior changes.

Prioritize files currently reported by `just lint-module-size`, then consider the remaining large modules if extraction is straightforward.

## Acceptance Notes

- An ExecPlan scopes the extraction order and validation strategy before code changes.
- Inline tests are moved to sibling test modules without behavior changes.
- `cargo check --tests` passes after each meaningful batch.
- Relevant targeted tests pass for each moved module.
- `cargo fmt`, `just lint-module-size`, and `just ci` pass before handoff.
