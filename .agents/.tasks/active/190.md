# Extract Duplicate Mutation Guard From Agent Module

**Created:** 2026-05-29
**Source:** Review of commit `b4e765140df9`

## Problem

`src/clients/agent.rs` is 2,696 lines and continues to accumulate responsibilities. Task 163 added the `ScheduledToolPlan` type, duplicate-mutation rejection helper, duplicate rejection output formatting, and duplicate-guard tests directly in `agent.rs`.

The file-size concern is pre-existing, but the duplicate guard is a cohesive ownership boundary that can be extracted without a broad agent-loop refactor.

## Relevant Files

- `src/clients/agent.rs`
- Potential new module under `src/clients/agent/` or another locally consistent agent submodule location
- `src/clients/tools/mod.rs`
- `src/clients/tools/edit.rs`
- `src/clients/tools/write.rs`

## Fix Direction

Move duplicate-mutation scheduling logic out of `agent.rs` into a focused module. The extracted module should own the scheduled plan wrapper, duplicate path tracking, rejection output construction, and the focused duplicate-guard tests where practical.

Keep the public surface small and avoid changing agent-loop behavior. If module layout conflicts with the existing `src/clients/agent.rs` file layout, use the smallest Rust module structure that fits the current crate organization.

## Acceptance Notes

- `agent.rs` no longer owns the duplicate guard implementation details.
- Existing duplicate guard behavior and tests continue to pass.
- The extraction does not expand the agent API surface more than necessary.
- Run `cargo test duplicate_guard`, `cargo check --tests`, `cargo fmt`, `just coverage-check`, and the required final verification for code changes.
