# Decouple Turn Counting From Usage Reporting

**Created:** 2026-06-09
**Source:** Code review of `src/clients/agent_state.rs` (2026-06-09)

## Problem

`accumulate_usage` (`src/clients/agent_state.rs:106`) increments `turn_count`
only inside `if let Some(usage) = turn_usage`. Against a provider that omits
the `usage` object, completed turns are never counted:

- The done summary and `--output-format json` report `0 turns` for a
  multi-turn run.
- Telemetry `turn_index` (derived as `turn_count + 1` in
  `src/clients/agent/agent_loop.rs:248`) stays frozen at 1, so every API
  attempt and tool call is attributed to the same turn.

Turn counting is a fact about loop iterations, not about whether the provider
reported token usage.

## Relevant Files

- `src/clients/agent_state.rs` — `accumulate_usage`
- `src/clients/agent.rs` — `accumulate_usage` wrapper, `turn_count` accessor
- `src/clients/agent/agent_loop.rs` — `complete_turn` telemetry `turn_index`

## Fix Direction

Increment `turn_count` once per completed API turn unconditionally (in the
loop, after `complete_turn` succeeds), and accumulate usage separately when
present. Verify telemetry `turn_index` still reflects the upcoming turn.

## Acceptance Notes

- A multi-turn run against a backend that returns no `usage` reports the
  correct turn count in the done summary, JSON output, and task completion
  record.
- Telemetry `turn_index` advances per turn regardless of usage presence.
- Unit test covers a turn result with `usage: None`.
- `just ci` passes.
