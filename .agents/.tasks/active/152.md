# Prevent shell command-substitution in Bash-tool git commit messages

## Problem

When the agent runs `git commit -m "..."` via the Bash tool and the message contains unescaped backticks (e.g. for inline code spans like `` `Runtime::new()` ``), bash inside the double-quoted `-m` argument treats each backtick pair as command substitution. The substituted commands fail with `syntax error` or `command not found`, and any output (usually empty) is spliced into the commit message in place of the intended text.

## Evidence

Session `a301ac3a-a30e-4f72-9c08-a52863dc3efa`, call `call_39fd`:

```
bash: command substitution: line 0: `async fn main()'
bash: .await: command not found
```

The recorded commit body (commit `77562a7`, since amended) read:

```
Switch from  +  to  on
. No custom runtime configuration ...
```

— every backticked token was silently dropped. The commit still landed because the substitutions returned empty strings rather than non-zero exits that aborted `git commit`.

## Fix direction

Pick one or combine:

1. **Bash tool guidance** — Add a system-prompt / tool-doc note: when building `git commit -m` messages that include backticks or `$`, use `git commit -F -` with a heredoc (or single-quoted strings) instead of double-quoted `-m`.
2. **Bash tool safeguard** — In the Bash tool wrapper, detect `git commit -m "..."` arguments that contain unescaped backticks or unquoted `$(` and either reject the call with a clear error, or transparently rewrite to `git commit -F -` via stdin.
3. **Helper recipe** — Offer a `just commit` or similar helper that reads a multiline message from a file/stdin so the tool never has to embed it in a shell-double-quoted string.

Decide which combination to apply (instructions vs. tool-level enforcement) based on how often this pattern shows up in sessions.

## Acceptance

- Pick an approach and apply it.
- An agent following only the system prompt / tool docs writes a commit body containing inline code spans whose backticks survive verbatim in `git log`.
- If the Bash tool gains a safeguard, add a regression test that exercises a commit message with backticks.
