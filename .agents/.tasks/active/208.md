# Add Conversation-Level Context Window Management

**Created:** 2026-06-09
**Source:** Code review of the agent loop and retry overrides (2026-06-09)

## Problem

Conversation history grows without bound and is resent in full every turn
(`ConversationState` in `src/clients/agent_state.rs`). The only overflow
mitigation is the one-shot context-overflow retry that reduces
`max_output_tokens` (`RequestOverrides.context_overflow_retry_used` in
`src/clients/retry.rs`) — which cannot help when the *input* itself exceeds
the model's context window. Long-running tasks and long `--continue` chains
will eventually hard-fail with no recourse and no guidance.

Task 146 (blocked) addresses one contributor — large tool outputs — but not
conversation-level growth or the failure mode when the window is exhausted.

## Relevant Files

- `src/clients/agent_state.rs` — `ConversationState`
- `src/clients/agent/agent_loop.rs` — turn loop, history passed per request
- `src/clients/retry.rs` — context-overflow classification and overrides
- Task 146 — tool-output compaction (related, narrower)
- Session format docs — compaction interacts with persisted history

## Fix Direction

Decide on a strategy and record it in an ADR; candidates in increasing
ambition:

1. Graceful exhaustion: detect input-too-large overflow distinctly and fail
   with actionable guidance (e.g., suggest `--fork` or a fresh session)
   instead of a raw API error.
2. Bounded history: drop or truncate oldest tool outputs first (persisted
   session keeps full fidelity; only the model-facing history is compacted).
3. Summarization-based compaction of older turns.

This affects the agent loop and session semantics; per AGENTS.md it is L work
requiring an ExecPlan before implementation.

## Acceptance Notes

- A conversation that exceeds the context window no longer dies with a raw
  provider error; behavior matches the documented strategy.
- Persisted session files remain a faithful record even if model-facing
  history is compacted.
- Tests cover the overflow-handling path with the stub backend.
- ExecPlan and ADR exist before implementation.
- `just ci` passes.
