# Add Structured Bulk Edit Support for Mechanical Transformations

**Status:** Pending
**Priority:** P2
**Effort:** M
**Created:** 2026-05-15
**Source:** Edit tool session analysis across recent persisted sessions
**ExecPlan:** -
**Depends on:** -

## Problem

Literal `Edit` is awkward for mechanical transformations. Agents frequently need to remove many repeated modifiers, fixture fields, task rows, or declarations. Those sessions produce many similar edits, occasional max-edit failures, and substantial read/search/retry work.

Representative examples:

- Session `ffacd995-6286-4d1c-8261-a7a9775b13c4`, line 387: an 18-edit `source/terminal/keys.ts` export-removal call exceeded the max of 10.
- Session `ffacd995-6286-4d1c-8261-a7a9775b13c4`, lines 391 and 394: the agent split the same mechanical operation into 10-edit and 8-edit calls.
- Session `cffbf360-ccae-450f-8026-d534709f4cb0`, lines 230-297: regex/script-style repair became more practical than repeated literal edits for fixture migration fallout.

## Relevant Files

- `src/clients/tools/edit.rs`
- Tool definitions and prompt text that describe editing capabilities
- Any tests for tool schemas or tool execution

## Fix Direction

Design and add a structured bulk-edit capability for safe mechanical transformations. Possible shapes include:

- A separate tool for constrained text transforms.
- An extension to `Edit` for repeated declaration/modifier removal.
- A helper mode that applies a limited literal replacement across all unique matches with a bounded diff summary.

Avoid an unconstrained arbitrary regex replacement unless its safety story is clear.

## Acceptance Notes

- The implementation supports at least one recurring mechanical case from session evidence, such as removing repeated `export ` or repeated field lines.
- The operation has clear safety constraints and bounded output.
- Tests cover multiple replacements in one file and a case where the transform should refuse to proceed.
- The tool description clearly distinguishes this from literal single-location `Edit`.
