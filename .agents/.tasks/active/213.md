# Support .worktreeinclude

## Summary

Add support for a `.worktreeinclude` file in the project root that automatically copies gitignored files (e.g. `.env`, `.env.local`, `config/secrets.json`) when creating a worktree via `cake worktree`.

## Problem

When using worktrees, gitignored files like `.env` and secrets don't exist in the new worktree, so the environment isn't ready. Users have to manually copy these files after every `cake worktree` command.

## Relevant Files

- `src/cli/` — likely where worktree-related CLI code lives
- `src/worktree/` or similar module for worktree operations
- Documentation for the worktree subcommand

## Fix Direction

Introduce a `.worktreeinclude` file in the project root (gitignored-friendly) that lists glob patterns of files to copy into new worktrees. When `cake worktree` is run, it reads this file and copies matching files from the source worktree to the new one. Follow `git worktree add` semantics for where files live.

## Acceptance Notes

- [ ] `cake worktree <name>` reads `.worktreeinclude` from the source project root if it exists
- [ ] Files matching the glob patterns in `.worktreeinclude` are copied to the new worktree
- [ ] Sensible error handling if a listed file doesn't exist in the source
- [ ] Document the `.worktreeinclude` format in help text
- [ ] Existing worktree tests continue to pass
- [ ] Run `cargo fmt` and `just ci` before handoff
