# Implement User-Defined Toolbox Tools

**Source:** Active ExecPlan `toolboxes-plan.md`

**Priority:** P2

**Effort:** XL

**ExecPlan:** `.agents/exec-plans/active/toolboxes-plan.md`

**Depends on:** 050 (Backend abstraction, completed), 051 (Tool abstraction, completed)

**Problem:** Cake has four hardcoded tools (Bash, Read, Edit, Write). Users cannot extend the tool set with custom executables. Other agent CLIs support user-defined tools via a simple stdin/stdout protocol.

**Files involved:** `src/config/toolbox.rs` (new), `src/clients/tools/toolbox.rs` (new), `src/clients/tools/mod.rs`, `src/clients/agent.rs`, `src/main.rs`

**Fix:**
1. **Discovery**: Scan `CAKE_TOOLBOX` colon-separated directories (default `~/.config/cake/tools/`) for executable files, skipping hidden files, `.md`/`.txt`, non-executables, and directories.
2. **Describe protocol**: Run each executable with `TOOLBOX_ACTION=describe` to get name, description, and parameter schema. Support both JSON format (compact `args` and full `inputSchema`) and text format. Skip broken tools with a warning.
3. **Execute protocol**: Dispatch `tb__*` tool calls by running the executable with `TOOLBOX_ACTION=execute`, passing arguments on stdin, capturing stdout, and checking exit codes.
4. **Agent integration**: Accept toolbox tools in `Agent::new()`, store a registry, and dispatch `tb__*` calls from `execute_tool`.
5. **CLI**: Add `--toolbox <DIR>` flag and optional `cake tools` subcommand (list, show, use, make).

Toolbox tools run as separate processes without sandboxing — they are user-provided and trusted.
