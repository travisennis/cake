## Fix Direction

Add guidance for cfg-sensitive changes:

- If changed code includes `#[cfg]`, platform-specific modules, or target-specific behavior, run the narrowest feasible target checks for installed targets.
- If a target is unavailable, explicitly report the gap in the task notes and final response.
- For Linux-specific Rust modules on macOS, prefer `cargo check --target x86_64-unknown-linux-gnu` when that target is installed and dependencies allow it.

Keep the guidance pragmatic; not every task needs exhaustive cross-compilation.
