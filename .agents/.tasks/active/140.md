# Harden Edit Schema Guidance Against Malformed Arguments

**Status:** Pending
**Priority:** P2
**Effort:** S
**Created:** 2026-05-15
**Source:** Edit tool session analysis across recent persisted sessions
**ExecPlan:** -
**Depends on:** -

## Problem

Recent sessions included malformed `Edit` calls with missing or duplicate fields. These are agent-call mistakes, but the tool schema and descriptions should make the correct shape harder to miss.

Observed failures across persisted sessions include:

- Missing `old_text`: 4 calls.
- Missing `new_text`: 2 calls.
- Missing `path`: 2 calls.
- Missing `edits`: 1 call.
- Duplicate `new_text`: 1 call.

Representative examples:

- Session `a7e8b7d5-b223-4ba4-b156-6d123ed039ae`, line 80: missing `old_text`.
- Session `851f7c6f-6a99-4023-8e82-b0579945c79f`, line 131: missing `new_text`.
- Session `8ea24946-26da-401c-8ec9-a3751fe7653b`, line 41: missing `path`.
- Session `b040b7c9-3234-42dc-b52a-2078ef755b11`, line 65: duplicate `new_text`.

## Relevant Files

- `src/clients/tools/edit.rs`
- Tool schema snapshot tests, if present
- Prompt/tool-description code that exposes Edit to models

## Fix Direction

Tighten the tool schema and description so agents reliably send:

- Top-level `path`.
- Top-level `edits`.
- Per-edit `old_text`.
- Per-edit `new_text`.

Consider adding examples in the description, clearer required-field wording, or stronger validation messages that quote the expected minimal shape.

## Acceptance Notes

- The schema still requires `path`, `edits`, `old_text`, and `new_text`.
- Invalid-argument errors include the expected shape or enough context to self-correct quickly.
- Tests cover at least one malformed-argument path if the validation layer is locally testable.
- No unrelated tool schemas change.
