# Add `--json` Flag to `cake debug models`

## Problem

`cake debug models` currently outputs a formatted ASCII table. There's no way
to get the model definitions as JSON for programmatic consumption (piping to
`jq`, scripting, tooling). Since `ModelDefinition` already derives `Serialize`,
the data is ready — it just needs a JSON output path.

## Relevant Files

- `src/cli/debug.rs` — the single file that needs changes

## Implementation Notes

Add a `--json` flag to the `Models` subcommand:

```rust
enum DebugSubcommand {
    Models {
        #[arg(long, default_value_t = false)]
        json: bool,
    },
}
```

When `--json` is passed:

1. Collect the model map values into a `Vec<&ModelDefinition>`, sorted by name
   (same sort order as the table).
2. Serialize as a JSON array using `serde_json::to_string_pretty`.
3. Print to stdout.

When `--json` is not passed, the existing table path runs unchanged.

Key details:

- Use the full `ModelDefinition` struct — consumers expect all fields (not
  just the subset shown in the table), including `provider`, `temperature`,
  `providers`, etc.
- `ModelDefinition` already derives `Serialize` — no derive changes needed.
- Do not leak API keys: the JSON includes `api_key_env` (the env var name),
  not the resolved API key value, matching the existing table.
- No new dependencies.
- Add tests for JSON output (empty map, non-empty map, sorted order).

## Acceptance Notes

- [ ] `cake debug models` (no flag) still prints the existing table unchanged.
- [ ] `cake debug models --json` prints a JSON array of model definitions

- [ ] `cake debug models --json` with no models configured prints `[]`.
- [ ] Existing tests continue to pass.
- [ ] New tests cover JSON output for empty and non-empty model maps.
