## Problem

For small changes, the preferred flow is focused validation while iterating, then one final `just ci` after all code, task, docs, or config edits are complete.

## Fix Direction

Update the relevant guidance so agents prefer focused checks during implementation and deslop passes, then save the required full `just ci` for the final post-edit state when code, config, or dependency files changed.

Keep the existing requirement to run full CI before finishing code/config/dependency changes.
