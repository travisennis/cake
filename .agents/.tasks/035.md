# Retire Bash Shell Parser Expansion

**Source:** prompt.md item 32 (also covers data-context follow-up from items 31/75)

**Priority:** P0

**Effort:** S

**Depends on:** 033 (already complete), 034

**Problem:** This task originally proposed replacing the Bash destructive-command guard with real shell parsing so it could safely distinguish executable shell syntax from literal data. That direction no longer matches the safety decision in task 034.

Task 033 already removed the blanket data-context skip and added command-substitution inspection. Current `bash_safety.rs` also strips quoted literal data before matching, so examples such as `git commit -m "docs: explain git reset --hard"` are not wrongly blocked. A full parser-backed policy engine would create the wrong expectation that this layer is a shell security boundary.

**Files involved:** `src/clients/tools/bash_safety.rs`

**Decision:** Do not add a shell parser or broaden `bash_safety.rs` into a bypass-prevention engine. Keep the preflight layer as a narrow best-effort destructive-operation guard, and rely on the OS sandbox for filesystem enforcement.

**Fix:**
1. Remove or revise any comments/docs that imply the Bash preflight guard fully parses shell syntax or provides security enforcement.
2. Keep existing regression tests for literal quoted data versus executable substitutions.
3. Add only bounded tests for documented wrapper support (`bash -c` and `sh -c`) if coverage is missing.
4. Do not add support for extra shells or forms such as `zsh -c`, `sh -lc`, heredoc parsing, or complete shell tokenization unless a separate product requirement justifies it.

**Acceptance criteria:**

- No new shell-parser dependency is introduced for `bash_safety.rs`.
- The task text and code comments align with task 034's boundary: preflight is best-effort, sandbox is enforcement.
- Tests continue to cover quoted literal data, command substitutions, and documented `bash -c` / `sh -c` wrappers.
- Any remaining limitations are documented as intentional scope, not parser bugs.

**Status:** Completed
