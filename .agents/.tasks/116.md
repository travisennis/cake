# Eliminate Redundant `num_turns` / `turn_count` in `task_complete`; Add Tool-Call Counter

- **Status:** Open
- **Created:** 2026-05-11
- **Priority:** P3
- **Effort:** M

## Problem

The `task_complete` event in the session JSONL has both `turn_count` and `num_turns` fields, but they always hold the same value (set identically in `emit_task_complete_record`):

```rust
// src/clients/agent.rs:387-399
pub fn emit_task_complete_record(&mut self, outcome: TaskOutcome, duration_ms: u64) -> ... {
    let record = StreamRecord::TaskComplete(TaskCompleteData {
        turn_count: self.turn_count,
        num_turns: self.turn_count,   // identical
        ...
    });
}
```

The design docs say `num_turns` is "Alias of `turn_count` retained for consumer compatibility." There is no consumer using this differently; the field is dead weight.

The agent also lacks a counter for how many tool calls were executed during the task. This is useful for analyzing session behavior.

## Proposed Fix

1. Keep `turn_count` as the API-turn counter (what it currently counts).
2. Replace `num_turns` with `tool_call_count` (or similar), which counts the number of tool calls executed.
3. Add a `tool_call_count: u32` field to `Agent`, incremented each time a tool is executed in the agent loop.
4. Populate `tool_call_count` in the `TaskCompleteData` struct instead of the duplicated `num_turns`.
5. Remove the design-doc text about `num_turns` being an alias.
6. Update all tests, snapshots, and the session-record test helpers (`task_complete()` in `src/config/session.rs`).

- The `task_complete` JSONL record no longer contains a `num_turns` field.
- A new `tool_call_count` field (name TBD) reports the number of tool executions during the task.
- `turn_count` continues to count API turns (unchanged semantics).
- All existing tests pass; the insta snapshot is updated.
