# Implement or Remove suppress_output Hook Field

**Parent:** 060

**Source:** prompt.md item 47

**Priority:** P1

**Effort:** S

**Problem:** `HookOutput` deserializes a `_suppress_output: Option<bool>` field that is then completely ignored. Either the field is part of the public hook contract (and should affect output) or it should not be deserialized at all. The current state lies to hook authors who set it expecting a behavior change.

**Files involved:** `src/hooks.rs`, hook docs

**Fix:** Pick one:

- **Implement it** — define exactly what `suppress_output: true` means (skip stream-json emission for this hook? skip session-record persistence? both?), wire it through, document it, and rename the field off the leading underscore.
- **Remove it** — delete the field from `HookOutput`, document that hooks cannot suppress output, and add a settings/serde test confirming a stray `suppress_output` key in user JSON is reported (loud) rather than silently ignored.

Either resolution must be reflected in the hooks documentation.

**Status:** Completed
