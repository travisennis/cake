# Document ResponsesApiInputItem Construction Boundary

**Source:** Review of commit `21fb640f74a3` (task 061)

**Priority:** P3

**Effort:** XS

**Problem:** `ResponsesApiInputItem<'a>` is `pub(super)` and is the typed Responses API request DTO, but its inner field types `ResponsesMessageContent<'a>` and `ResponsesReasoningSummary<'a>` have private fields with no constructor. The intent is that callers in `src/clients/responses.rs` build instances only through `ResponsesApiInputItem::from(&ConversationItem)`, never by hand. A reader of `responses.rs` who tries to construct one directly will get a confusing privacy error rather than a clear signal of intent.

**Files involved:** `src/clients/types.rs`

**Fix:** Add a one- or two-line module/doc comment on `ResponsesApiInputItem`, `ResponsesMessageContent`, and `ResponsesReasoningSummary` explaining that they are constructed only via the `From<&ConversationItem>` impl in this module so the API wire shape is owned in one place. No code changes required beyond doc comments.

**Acceptance notes:** Doc comments exist on the three types describing the construction boundary. `just ci` passes.
