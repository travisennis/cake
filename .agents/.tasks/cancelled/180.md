# Clarify Commit/Task-Complete Ordering in Workflow Guidance

**Created:** 2026-05-24
**Source:** Evaluation of session `98433c0e-2228-4d40-9e88-234141f16fc1`

## Problem

The AGENTS.md task queue rules state:

> "When a task includes committing and task-status updates, commit the
> intended code/task changes together unless the user asks for separate
> commits."

This guidance says to commit everything together, but does not specify the
ordering of `ahm task complete <id>` relative to the final `git commit`.
Since `ahm task complete` modifies task files (moves them, regenerates
indexes), the timing matters:

- **Commit first, then task-complete**: Leaves the task-file changes
  uncommitted, creating a second commit and violating "commit together."
- **Task-complete first, then commit**: Ensures all changes (code + task
  files) are in one commit, but the agent must remember to stage the moved
  task files alongside the source changes.

In session `98433c0e-2228-4d40-9e88-234141f16fc1`, the agent:

1. Committed source code changes only (`cbce92a`)
2. Ran `ahm task complete 177`, which modified task files
3. Committed the task-file changes separately (`8f3a40e`)
4. Realized this violated the "commit together" rule
5. Ran `git reset --soft HEAD~2` to undo both commits
6. Staged both source and task files together
7. Made a single combined commit (`7eec0c4`)

This consumed ~3 extra reasoning turns and ~6 git operations that would not
have been needed with clearer ordering guidance.

## Recommendation

Update the AGENTS.md task queue rules to clarify the ordering:

> **Workflow**: When committing and marking a task complete:
> 1. First run `ahm task complete <id>` (or `ahm task cancel <id>`).
>    This moves the task file and regenerates indexes.
> 2. Then stage all changed files — source code, moved task files,
>    regenerated indexes — together with `git add`.
> 3. Finally make a single commit with all changes.

This ensures the agent always ends with one commit containing everything
that changed, without needing to backtrack.
