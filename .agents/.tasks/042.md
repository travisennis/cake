# Stop Silently Truncating Responses History

**Source:** prompt.md item 44

**Priority:** P0

**Effort:** S

**Problem:** `extract_instructions` silently drops all conversation items after a misplaced system message. That can amputate session history and change model behavior without a hard failure.

**Files involved:** `src/clients/responses.rs`

**Fix:** Return an error for invalid system-message placement or strip only the offending item with a clear warning. Add regression tests for mid-history system messages.

**Status:** Completed
