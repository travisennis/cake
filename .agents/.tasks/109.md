# 109 — Edit tool: CRLF replacement text double-encoding

Status: Open
Created: 2026-05-06

## Summary

`normalize_replacement_line_endings` in `src/clients/tools/edit.rs` converts
LF line endings in replacement text to CRLF when the target file uses CRLF
line endings. It uses `content.replace('\n', "\r\n")`, which will turn an
existing `\r\n` in the replacement text into `\r\r\n` (double-encoding).

## Where

`src/clients/tools/edit.rs`, `normalize_replacement_line_endings`:

```rust
fn normalize_replacement_line_endings(content: &str, ending: LineEnding) -> String {
    match ending {
        LineEnding::Crlf => content.replace('\n', "\r\n"),
        LineEnding::Lf => content.to_string(),
    }
}
```

## Risk

Low. LLMs nearly always emit `\n` line endings, so the replacement text will
not contain `\r\n` in practice. The old code had the same issue (it restored
line endings post-hoc with `content.replace('\n', "\r\n")` on the entire
file), so this is not a regression.

## Fix direction

Replace only bare `\n` bytes (those not already preceded by `\r`). Something
like scanning character-by-character or using a regex that only matches `\n`
not immediately following `\r`.
