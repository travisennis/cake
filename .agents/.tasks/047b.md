**Parent:** 047

**Source:** prompt.md items 1 and 62

**Fix:**
1. Extend the `Tool` trait (from 051) so `execute` receives `&ToolContext`.
2. Update `validate_path`, `validate_path_for_write`, `cached_cwd`, `cached_temp_dirs`, `get_additional_dirs`, `get_skill_dirs`, `get_settings_dirs` to take a `&ToolContext` argument. Their bodies read from the context, not from globals.
3. Update sandbox profile construction (Seatbelt, Landlock) to take a `&ToolContext`.
4. `Agent` holds an `Arc<ToolContext>` and passes it into each tool execution.
5. Existing tests should continue to pass; add tests that two `ToolContext`s with different `additional_dirs` produce different validation outcomes when used concurrently.
