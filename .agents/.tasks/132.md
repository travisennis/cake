## Problem

The hook that provides Rust workflow failure guidance is intended to be silent unless the relevant command fails. In session `3e0e3899-b7b6-4ac9-8d11-ea93a26bffc2`, successful commands still received failure-oriented additional context.

Observed examples:

- `just ci 2>&1` returned `[exit:0]` and `All checks passed!`, but the output included `Additional hook context: A Rust workflow command failed in the cake repo...`.
- The final `git commit` succeeded, but the output still included the same failure guidance.

This can mislead the model into thinking a successful verification command failed, and it adds avoidable noise to tool results.

## Relevant Files

- `.cake/hooks.json`
- Hook implementation files under `src/`
- Any hook script or configured command that emits `A Rust workflow command failed in the cake repo`

## Fix Direction

Find the hook or hook command that emits the Rust workflow failure guidance. Ensure it emits no additional context when the inspected tool command succeeds. The intended behavior is silence on success and guidance only on error.

Check both the hook decision path and the logic that appends `additional_context` to tool outputs.

## Acceptance Notes

- Successful Rust workflow commands such as `just ci` do not receive the failure guidance context.
- Failed Rust workflow commands still receive the actionable guidance.
- Tests or focused manual verification cover success and failure cases.
- `just ci` passes after the change.
