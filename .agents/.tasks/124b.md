## Summary

`src/clients/tools/bash.rs` has a top-of-file `#![expect(clippy::string_slice, reason = "string indexing is on ASCII byte boundaries derived from UTF-8 aware helpers")]`. Replace with localized per-site `#[expect]` attributes (or refactor to UTF-8-safe alternatives).

## Files Involved

- `src/clients/tools/bash.rs`

## Fix Direction

1. Remove the module-level `#![expect(clippy::string_slice, ...)]`.
2. Audit each slice. The `truncate_output` helper already uses `floor_char_boundary` / `ceil_char_boundary`; nearby slice sites should follow the same pattern.
3. Where a slice cannot be safely refactored, wrap in a narrow `#[expect(clippy::string_slice, reason = "...")]` block naming the specific invariant.
4. Confirm `just clippy-strict` is clean.

## Acceptance Notes

- Module-level attribute removed.
- `truncate_output` now uses `split_at` with precomputed UTF-8 boundaries instead of direct string slicing.
